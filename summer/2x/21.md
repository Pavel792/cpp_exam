# 21. Использование шаблонов

## Передача функторов без `std::function`

**Функтор** — объект с перегруженным `operator()`:

```cpp
struct Adder {
    int n;
    int operator()(int x) { return x + n; }
};

Adder add5{5};
add5(3);  // вызывает operator() → 8
```

**`std::function`** — обёртка которая хранит любой callable с одинаковой сигнатурой:

```cpp
std::function<int(int)> f;
f = Adder{5};                      // функтор
f = [](int x){ return x; };       // лямбда
f = some_function;                 // обычная функция
```

Удобно, но медленно — внутри type erasure и виртуальные вызовы.

### Как поле класса

```cpp
// с std::function — можно менять callback в рантайме
struct Button {
    std::function<void()> on_click;
};

Button b;
b.on_click = []{ std::cout << "сохранить"; };
b.on_click = []{ std::cout << "удалить"; };  // ✅ меняем спокойно
```

```cpp
// без std::function — шаблонный класс, тип фиксируется при создании
template<typename F>
struct Button {
    F on_click;
    void click() { on_click(); }
};

Button b{[]{ std::cout << "нажали"; }};
b.click();
// b.on_click = другая_лямбда;  ❌ — у каждой лямбды свой уникальный тип
```

### Как аргумент функции

```cpp
// с std::function — медленно
void apply(std::function<int(int)> f, int x) { f(x); }

// без std::function — шаблонный параметр, компилятор инлайнит
template<typename F>
void apply(F f, int x) { f(x); }

apply([](int x){ return x + 1; }, 5);  // ✅
apply(Adder{5}, 5);                     // ✅
```

| | шаблон | `std::function` |
|---|---|---|
| Скорость | ✅ быстро | ❌ медленно |
| Менять callback | ❌ нельзя | ✅ можно |
| Тип фиксирован | да, при создании | нет |

## Шаблоны функций и автовывод параметров

```cpp
template<typename T>
T add(T a, T b) { return a + b; }

add(1, 2);       // T=int — выведено автоматически
add(1.0, 2.0);   // T=double — выведено автоматически
add<int>(1, 2);  // T=int — указано явно
```

Вывод не работает для возвращаемого типа — нужно указывать явно:

```cpp
template<typename T>
T foo() { return T{}; }

foo();       // ❌ — компилятор не знает T
foo<int>();  // ✅
```

### Несколько параметров

```cpp
template<typename A, typename B>
auto add(A a, B b) -> decltype(a + b) {
    return a + b;
}

add(1, 2.0);  // A=int, B=double, результат double
```

### Частичное указание параметров

```cpp
template<typename R, typename A, typename B>
R add(A a, B b) { return a + b; }

add<double>(1, 2);  // R=double явно, A=int и B=int выведены из аргументов
```

## CTAD — Class Template Argument Deduction

До C++17 параметры шаблонного класса нужно было писать явно. С C++17 компилятор выводит их из конструктора:

```cpp
std::pair<int, double> p{1, 2.0};  // до C++17
std::pair p{1, 2.0};               // C++17 — T выводится автоматически
```

### При инициализации переменных

```cpp
std::vector v{1, 2, 3};       // std::vector<int>
std::pair p{"hello", 42};     // std::pair<const char*, int>
std::tuple t{1, 2.0, "str"};  // std::tuple<int, double, const char*>
```

### При создании временных объектов

```cpp
template<typename T>
struct Foo {
    T value;
    Foo(T v) : value(v) {}
};

auto f = Foo{42};   // временный объект, T=int
void bar(Foo<int>);
bar(Foo{42});       // временный объект передаётся в функцию
```

### Deduction guides

Если компилятор не может вывести параметры сам:

```cpp
template<typename T>
struct Foo { Foo(T* ptr) {} };

template<typename T>
Foo(T*) -> Foo<T>;  // подсказка компилятору

Foo p{new int{5}};  // T=int
```

## Шаблонные методы и конструкторы

### В обычном классе

```cpp
struct Converter {
    template<typename T>
    T convert(int x) { return static_cast<T>(x); }
};

Converter c;
c.convert<double>(42);  // T не выводится из возвращаемого — указываем явно
```

### Шаблонный конструктор

```cpp
struct Foo {
    template<typename T>
    Foo(T value) { std::cout << value; }
};

Foo f1{42};      // T=int — выводится из аргумента
Foo f2{3.14};    // T=double
```

### В шаблонном классе

```cpp
template<typename T>
struct Foo {
    template<typename U>
    void bar(U x) {
        // T — параметр класса, U — параметр метода
    }
};

Foo<int> f{42};
f.bar(3.14);  // T=int, U=double
```

### Определение вне класса — два `template`

```cpp
template<typename T>  // параметр класса
template<typename U>  // параметр метода
void Foo<T>::bar(U x) { }  // нельзя объединить в один template<>
```

## Шаблонные лямбды

### До C++20 — `auto` в параметрах

```cpp
auto lam = [](auto x) { return x + 1; };

lam(42);    // x=int
lam(3.14);  // x=double
```

### C++20 — явный шаблонный параметр

```cpp
auto lam = []<typename T>(T x) { return x + 1; };
lam<double>(42);  // T=double явно
```

Явный параметр нужен чтобы связать типы нескольких параметров:

```cpp
// auto — x и y могут быть разных типов
auto lam1 = [](auto x, auto y) { return x + y; };

// шаблонный параметр — x и y одного типа
auto lam2 = []<typename T>(T x, T y) { return x + y; };

lam1(1, 2.0);  // ✅ int и double
lam2(1, 2.0);  // ❌ T не может быть одновременно int и double
lam2(1, 2);    // ✅ оба int
```

## Проблемы с `>>` до C++11

```cpp
std::vector<std::vector<int>> v;   // ❌ до C++11 — >> читается как оператор сдвига
std::vector<std::vector<int> > v;  // ✅ до C++11 — нужен пробел
std::vector<std::vector<int>> v;   // ✅ C++11 — починили
```

## Псевдонимы типов: `template<> using`

```cpp
// обычный using — конкретный тип
using IntVector = std::vector<int>;

// шаблонный using — alias template
template<typename T>
using Vector = std::vector<T>;

Vector<int> v;     // то же что std::vector<int>
Vector<double> u;  // то же что std::vector<double>
```

Фиксирование части параметров:

```cpp
template<typename T>
using StringMap = std::map<std::string, T>;

StringMap<int> m;  // то же что std::map<std::string, int>
```

`typedef` не умеет шаблоны — только `using`:

```cpp
template<typename T>
typedef std::vector<T> Vector;  // ❌ — typedef не поддерживает шаблоны

template<typename T>
using Vector = std::vector<T>;  // ✅
```

`Vector<int>` и `std::vector<int>` — один и тот же тип, просто псевдоним.

## Шаблоны переменных (C++17)

Значение переменной зависит от типа:

```cpp
template<typename T>
constexpr T pi = 3.14159265358979;

pi<float>;   // float точность
pi<double>;  // double точность
```

### Примеры из стандартной библиотеки

Все `_v` суффиксы — шаблоны переменных поверх шаблонных структур:

```cpp
std::is_integral_v<int>;     // true  (вместо std::is_integral<int>::value)
std::is_integral_v<double>;  // false
std::is_same_v<int, int>;    // true
std::is_same_v<int, double>; // false

// внутри стандартной библиотеки примерно так:
template<typename T>
constexpr bool is_integral_v = std::is_integral<T>::value;
```

## Переиспользование шаблонов между единицами трансляции

Шаблоны автоматически `inline` — ODR не нарушается при определении в заголовке.

### Обычный способ — определение в заголовке

```cpp
// header.h
template<typename T>
void foo(T x) { ... }  // каждый .cpp инстанцирует сам
```

Плюс — работает с любым `T`. Минус — каждый `.cpp` компилирует шаблон заново.

### Явное инстанцирование — определение в `.cpp`

```cpp
// header.h
template<typename T>
void foo(T x);  // только объявление

// foo.cpp
template<typename T>
void foo(T x) { ... }  // определение спрятано здесь

template void foo<int>();    // явно инстанцируем нужные типы
template void foo<double>();
```

```cpp
// file1.cpp
foo(1);   // ✅ линкер найдёт foo<int> из foo.cpp
foo('a'); // ❌ foo<char> не инстанцировали
```

Компилятор компилирует каждый `.cpp` в `.o`, линкер склеивает всё в программу — инстанцирование из `foo.cpp` доступно всем через линкер.

| | Определение в заголовке | Явное инстанцирование |
|---|---|---|
| Работает с любым T | ✅ | ❌ только указанные |
| Скорость компиляции | ❌ каждый .cpp компилирует | ✅ один раз |
| Удобство | ✅ | ❌ нужно добавлять типы вручную |

На практике почти всегда определяют в заголовке — гибкость важнее.
