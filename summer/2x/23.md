# 23. Вызов шаблонных функций

## Автовывод шаблонных параметров

Компилятор сам определяет типы параметров из аргументов:

```cpp
template<typename T>
void foo(T x) { }

foo(42);      // T=int
foo(3.14);    // T=double
foo<int>(42); // явно — но обычно не нужно
```

### С `const`

```cpp
template<typename T>
void foo(T x) { }

const int x = 5;
foo(x);  // T=int — const снимается при передаче по значению
```

Если параметр — ссылка, `const` сохраняется:

```cpp
template<typename T>
void foo(T& x) { }

const int x = 5;
foo(x);  // T=const int
```

### С ссылками

```cpp
template<typename T>
void foo(T& x) { }

int x = 5;
foo(x);  // T=int, параметр — int&
```

Universal reference — выводит категорию значения:

```cpp
template<typename T>
void foo(T&& x) { }

int x = 5;
foo(x);            // lvalue → T=int&,  параметр — int&
foo(5);            // rvalue → T=int,   параметр — int&&
foo(std::move(x)); // xvalue → T=int,   параметр — int&&
```

### С наследованием

```cpp
template<typename T>
void foo(T x) { }

struct Base {};
struct Derived : Base {};

Derived d;
foo(d);   // T=Derived — выводится точный тип, не Base

template<typename T>
void bar(T* x) { }

foo(&d);  // T=Derived
```

### Частичное указание параметров

Можно указать только часть — остальные выведутся:

```cpp
template<typename R, typename A, typename B>
R add(A a, B b) { return a + b; }

add<double>(1, 2);  // R=double явно, A=int и B=int выведены
```

## Невозможность автовывода

### Тип из `typename C::iterator`

```cpp
template<typename C>
void foo(typename C::iterator it) { }

std::vector<int> v;
foo(v.begin());                      // ❌ — нельзя вывести C из C::iterator
foo<std::vector<int>>(v.begin());    // ✅ — указываем явно
```

### При конфликтующих аргументах

```cpp
template<typename T>
void foo(T a, T b) { }

foo(1, 2.0);          // ❌ — T=int из первого, T=double из второго — конфликт
foo<double>(1, 2.0);  // ✅
```

### При необходимости пользовательских преобразований

```cpp
template<typename T>
void foo(T x, T y) { }

struct MyInt { MyInt(int) {} };

MyInt a(1);
foo(a, 42);  // ❌ — T=MyInt из первого, второй требует преобразования int→MyInt
             //       при выводе типа пользовательские преобразования не применяются
```

## Разрешение перегрузок с шаблонными функциями

### Термины

**Overload set** — все функции с одним именем которые видит компилятор.  
**Viable function** — функция из overload set, которая может быть вызвана с данными аргументами.  
**Best viable function** — наиболее подходящая из viable.

### Этапы

```
1. Построить overload set — все функции с этим именем
2. Выбрать viable — те что подходят по аргументам
3. Выбрать best viable — наиболее точное совпадение
4. Проверить доступ — public/private
```

### Пример

```cpp
void foo(int x) { std::cout << "int"; }
void foo(double x) { std::cout << "double"; }

template<typename T>
void foo(T x) { std::cout << "шаблон"; }

foo(42);    // best viable: foo(int) — точное совпадение
foo(3.14);  // best viable: foo(double) — точное совпадение
foo("hi");  // best viable: шаблон — нешаблонные не подходят
```

### Приоритет нешаблонных перегрузок

При одинаковой точности совпадения нешаблонная функция побеждает:

```cpp
void foo(int x) { std::cout << "нешаблонная"; }

template<typename T>
void foo(T x) { std::cout << "шаблонная"; }

foo(42);  // "нешаблонная" — одинаково точно, но нешаблонная побеждает
```

### `foo()` vs `foo<>()`

```cpp
void foo(int x) { }        // обычная функция

template<typename T>
void foo(T x) { }          // шаблон

foo(42);   // рассматривает все перегрузки — выберет обычную
foo<>(42); // рассматривает только шаблонные — выберет шаблон
```

### Пример Димова-Абрамса — специализации не влияют на выбор перегрузки

```cpp
template<typename T>
void foo(T x) { std::cout << "A"; }        // шаблон A

template<typename T>
void foo(T* x) { std::cout << "B"; }       // шаблон B — точнее для указателей

template<>
void foo<int>(int x) { std::cout << "C"; } // специализация шаблона A для int

foo(new int{5});  // "B" — компилятор выбирает перегрузку B как более точную
                  // C — специализация A, не B
                  // после выбора B специализации A не смотрятся
```

Именно поэтому специализации функций опасны — они специализируют конкретную перегрузку.
Если компилятор выбрал другую перегрузку — специализация игнорируется.
Вместо специализаций функций лучше использовать перегрузки.
