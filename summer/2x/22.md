# 22. Специализации

## Зачем нужны специализации

Шаблон описывает общее поведение для любого `T`. Но иногда для конкретного типа нужно сделать по-другому.

Например `std::vector<bool>` — обычный вектор хранит каждый элемент отдельно, но для `bool` эффективнее упаковывать биты:

```cpp
template<typename T>
struct vector {
    T* data;  // обычное хранение
};

template<>
struct vector<bool> {
    uint8_t* data;  // биты упакованы — специализация для bool
};
```

## Полные и частичные специализации шаблона класса

### Полная специализация — все параметры зафиксированы

```cpp
template<typename T>
struct Foo {
    void print() { std::cout << "общий"; }
};

template<>
struct Foo<int> {          // template<> — пустой список, все типы зафиксированы
    void print() { std::cout << "int"; }
};

Foo<double> a; a.print();  // "общий"
Foo<int> b;    b.print();  // "int"
```

Специализация по значению:

```cpp
template<int N>
struct Array { int data[N]; };

template<>
struct Array<0> { };  // особый случай для N=0
```

### Независимость специализаций

Полная специализация — полностью независимый класс. Можно добавлять и убирать любые поля и методы:

```cpp
template<typename T>
struct Foo {
    T value;
    void bar() {}
};

template<>
struct Foo<int> {
    int special_field;  // своё поле — не обязаны повторять value
    void baz() {}       // свой метод — bar необязателен
};
```

### Частичная специализация — часть параметров зафиксирована

```cpp
// общий шаблон
template<typename T, typename U>
struct Pair { T first; U second; };

// частичная специализация — оба параметра одного типа
template<typename T>
struct Pair<T, T> {
    T first; T second;
    bool equal() { return first == second; }
};

Pair<int, double> a;  // общий шаблон
Pair<int, int> b;     // частичная специализация
```

Специализация для указателей:

```cpp
template<typename T>
struct Foo { void print() { std::cout << "общий"; } };

template<typename T>
struct Foo<T*> {  // матчится только для указателей
    void print() { std::cout << "указатель"; }
};

Foo<int> a;   // общий, T=int
Foo<int*> b;  // специализация, T=int (не int*!)
```

## Метапрограммирование

Программирование на этапе компиляции — вычисления через шаблоны, результат известен до запуска программы.

### `is_void`

```cpp
template<typename T>
struct is_void { static constexpr bool value = false; };

template<>
struct is_void<void> { static constexpr bool value = true; };

is_void<int>::value;   // false
is_void<void>::value;  // true
```

### `is_same`

```cpp
template<typename T, typename U>
struct is_same { static constexpr bool value = false; };

template<typename T>
struct is_same<T, T> {  // частичная специализация — оба параметра одинаковые
    static constexpr bool value = true;
};

is_same<int, double>::value;  // false
is_same<int, int>::value;     // true
```

### `is_reference`

```cpp
template<typename T>
struct is_reference { static constexpr bool value = false; };

template<typename T>
struct is_reference<T&> { static constexpr bool value = true; };

template<typename T>
struct is_reference<T&&> { static constexpr bool value = true; };

is_reference<int>::value;    // false
is_reference<int&>::value;   // true
is_reference<int&&>::value;  // true
```

### `conditional`

Выбирает один из двух типов в зависимости от условия — тернарный оператор для типов:

```cpp
template<bool Cond, typename T, typename F>
struct conditional { using type = T; };  // Cond=true → берём T

template<typename T, typename F>
struct conditional<false, T, F> { using type = F; };  // Cond=false → берём F

conditional<true,  int, double>::type;  // int
conditional<false, int, double>::type;  // double
```

## Полные специализации функций

```cpp
template<typename T>
void foo(T x) { std::cout << "общий"; }

template<>
void foo<int>(int x) { std::cout << "int"; }  // тип можно не писать — выведется

foo(1);    // "int"
foo(1.0);  // "общий"
```

### Ограничение — нет частичных специализаций

```cpp
template<typename T>
struct Foo<T*> { };     // ✅ частичная специализация класса

template<typename T>
void foo<T*>(T* x) { } // ❌ частичная специализация функции — запрещена
```

### Специализации не участвуют в перегрузке

Компилятор сначала выбирает перегрузку, потом смотрит специализации:

```cpp
template<typename T>
void foo(T x) { std::cout << "шаблон"; }

template<>
void foo<int*>(int* x) { std::cout << "специализация int*"; }

template<typename T>
void foo(T* x) { std::cout << "шаблон для указателей"; }

foo(new int{5});  // "шаблон для указателей" — перегрузка 2 точнее
                  // специализация перегрузки 1 даже не рассматривалась!
```

Поэтому для функций вместо специализаций лучше использовать перегрузки.

### Эмуляция частичной специализации через структуры

```cpp
template<typename T>
struct FooImpl {
    static void run(T x) { std::cout << "общий"; }
};

template<typename T>
struct FooImpl<T*> {  // частичная специализация структуры — ✅
    static void run(T* x) { std::cout << "указатель"; }
};

template<typename T>
void foo(T x) { FooImpl<T>::run(x); }  // функция делегирует в структуру

foo(42);       // "общий"
foo(new int);  // "указатель"
```

## `std::swap`, ADL и сторонние контейнеры

### Проблема

`std::swap` работает через move — три операции. Для своего класса можно написать эффективнее (например просто поменять указатели).

### Нельзя добавлять перегрузки в `namespace std`

```cpp
namespace std {
    void swap(MyClass& a, MyClass& b) { }  // ❌ UB, запрещено стандартом
}
```

### Нельзя частично специализировать функцию

```cpp
template<typename T>
void std::swap<MyClass<T>>(MyClass<T>&, MyClass<T>&) { }  // ❌
```

### Решение — своя функция + ADL

Пишем `swap` в своём namespace. ADL (Argument-Dependent Lookup) найдёт её автоматически — при вызове функции без namespace компилятор ищет её в namespace аргументов:

```cpp
namespace myns {

struct MyClass {
    int* data;
    void swap(MyClass& other) {
        std::swap(data, other.data);  // меняем только указатели — быстро
    }
};

void swap(MyClass& a, MyClass& b) {  // свободная функция в том же namespace
    a.swap(b);
}

}  // namespace myns

myns::MyClass a, b;
swap(a, b);  // ADL находит myns::swap ✅
```

### Правильный способ вызова swap

```cpp
template<typename T>
void foo(T& a, T& b) {
    using std::swap;  // fallback если своей swap нет
    swap(a, b);       // ADL найдёт свою версию если есть, иначе std::swap
}
```

`std::swap` явно — всегда стандартная версия, ADL не работает.  
`swap` с `using std::swap` — сначала ADL, потом fallback на `std::swap`.
