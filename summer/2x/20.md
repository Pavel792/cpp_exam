# 20. Шаблоны классов

## Синтаксис объявления и определения

```cpp
// объявление
template<typename T>
struct Foo;

// определение
template<typename T>
struct Foo {
    T value;
    void bar();
};
```

### `typename` vs `class`

Полностью взаимозаменяемы для параметров-типов:

```cpp
template<typename T> struct Foo {};  // одно и то же
template<class T>    struct Foo {};  // одно и то же
```

Единственное отличие — внутри шаблона при обращении к зависимым типам только `typename`:

```cpp
template<typename T>
struct Foo {
    typename T::iterator it;  // ✅
    class T::iterator it;     // ❌ class здесь нельзя
};
```

### Параметры шаблона

Три вида:

```cpp
// 1. Тип
template<typename T>
struct Foo {};

// 2. Значение простого типа
template<int N>
struct Array {
    int data[N];
};
Array<10> a;  // N = 10 — известно на этапе компиляции

// 3. Шаблон-параметр (параметр сам является шаблоном)
template<template<typename> class Container>
struct Wrapper {
    Container<int> data;
};
Wrapper<std::vector> w;  // передаём сам шаблон, не инстанцирование
```

### Параметры по умолчанию

```cpp
template<typename T, typename Alloc = std::allocator<T>, int N = 10>
struct MyVector {};

MyVector<int> v;           // T=int, Alloc=std::allocator<int>, N=10
MyVector<int, MyAlloc> v;  // T=int, Alloc=MyAlloc, N=10
```

### Параметр без имени

Когда параметр нужен только для специализации или SFINAE:

```cpp
template<typename>   // параметр без имени
struct Foo {};
```

## `Foo` vs `Foo<T>`: когда что писать

`Foo` — это сам шаблон. `Foo<T>` — конкретное инстанцирование.

Внутри тела шаблонного класса `Foo` автоматически означает `Foo<T>`:

```cpp
template<typename T>
struct Foo {
    Foo* next;     // ✅ то же что Foo<T>* next
    Foo<T>* next;  // ✅ явно — то же самое
};
```

Снаружи класса — нужно писать `Foo<T>` явно:

```cpp
template<typename T>
struct Foo {
    Foo foo();
};

// снаружи — нужно и template<typename T> и Foo<T>
template<typename T>
Foo<T> Foo<T>::foo() { return *this; }

// с auto + trailing return type — чище
// после Foo<T>:: компилятор уже в скоупе Foo<T>, поэтому просто Foo
template<typename T>
auto Foo<T>::foo() -> Foo { return *this; }
```

## Инстанцирование

Шаблон — рецепт, инстанцирование — готовый класс по этому рецепту.
Реальный код появляется только когда компилятор подставляет конкретные параметры:

```cpp
template<typename T>
struct Foo { T value; };

Foo<int> a;     // компилятор генерирует Foo для T=int
Foo<double> b;  // компилятор генерирует Foo для T=double
```

### Независимость по методам

Методы инстанцируются независимо — только те, которые реально используются:

```cpp
template<typename T>
struct Foo {
    void bar() { T x; x.something(); }  // требует метод something()
    void baz() { T x; }                 // требует только конструктор
};

struct Simple {};  // нет метода something()

Foo<Simple> f;
f.baz();   // ✅ — baz инстанцируется, Simple подходит
f.bar();   // ❌ — только теперь компилятор проверит bar() и найдёт ошибку
```

### Когда происходит неявное инстанцирование

| Что | Когда инстанцируется |
|---|---|
| Поля класса | При инстанцировании класса |
| Нестатические методы | При вызове |
| Статические методы | При вызове |
| Статические поля | При использовании |
| Вложенные типы | При использовании |

### Incomplete types

Тип incomplete — если компилятор видел объявление, но не определение:

```cpp
struct Bar;      // incomplete
struct Bar { };  // complete
```

С incomplete типом можно если не нужен размер:

```cpp
template<typename T>
struct Node {
    T* ptr;   // ✅ указатель — размер T не нужен
    T& ref;   // ✅ ссылка — размер T не нужен
    T value;  // ❌ — нужен размер T
};

struct Bar;
Node<Bar> n;  // ✅ — Bar incomplete, но ptr/ref это допускают
```

### Явное инстанцирование

По умолчанию инстанцирование неявное. Можно потребовать явно:

```cpp
template struct Foo<int>;       // генерирует все методы
template void Foo<int>::bar();  // генерирует один метод
```

Зачем: если шаблон определён в `.cpp` — другие файлы не видят тело и не могут инстанцировать.
Явное инстанцирование в том же `.cpp` решает проблему:

```cpp
// foo.cpp
template<typename T>
void Foo<T>::bar() { ... }  // тело в .cpp

template struct Foo<int>;   // явно генерируем для int
// теперь другие .cpp могут использовать Foo<int>
```

Поэтому шаблоны почти всегда определяют в заголовочном файле — чтобы каждый `.cpp` видел тело.

## Независимость инстанциаций

`Foo<int>` и `Foo<double>` — полностью независимые типы. Никакой связи между ними нет.
Компилятор смотрит только на параметры шаблона, не на содержимое:

```cpp
template<typename T>
struct Foo {
    T value;
    static int count;  // у каждого инстанцирования СВОЯ статическая переменная
};

Foo<int>::count = 1;
Foo<double>::count = 2;

Foo<int>::count;    // 1
Foo<double>::count; // 2 — не связаны никак
```

Одно и то же инстанцирование — один класс, сколько бы раз ни использовалось:

```cpp
Foo<int> a;
Foo<int> b;
Foo<int> c;
// все три — один тип Foo<int>, один count
```

## Определение методов и статических членов снаружи класса

Каждое определение снаружи требует своего `template<typename T>`:

```cpp
template<typename T>
struct Foo {
    T value;
    void bar();
    static int count;
};

// метод снаружи
template<typename T>
void Foo<T>::bar() {
    // T здесь доступен
}

// статическое поле — нужно определить и инициализировать
template<typename T>
int Foo<T>::count = 0;
```
