# 70. Препроцессор: базовые макросы

## Конкатенация строковых литералов

Компилятор автоматически склеивает соседние строковые литералы в один:

```cpp
#include <iostream>

int main() {
    const char* s = "Hello, " "World" "!";  // склеивается в "Hello, World!"
    std::cout << s << "\n";
}
```

Используется в макросах для генерации строк:

```cpp
#define GREETING "Hello"
const char* full = GREETING ", World!";  // "Hello, World!"
```

## `#ifdef`, `#define`, ключ `-DDEBUG`

`#define DEBUG` без значения создаёт макрос, который просто **существует** — `#ifdef` проверяет факт существования, а не значение:

```cpp
#include <iostream>

#define DEBUG

#ifdef DEBUG
    void log(const char* msg) { std::cout << "[DEBUG] " << msg << "\n"; }
#else
    void log(const char* msg) {}  // пустая функция в релизе
#endif

int main() {
    log("приложение запущено");
}
```

Препроцессор решает какой блок оставить **до начала компиляции** — второй блок исчезает целиком, будто его не было:

```bash
g++ -E test.cpp   # покажет результат после препроцессора
```

Определять можно и через ключ компилятора, не трогая код:

```bash
g++ -DDEBUG main.cpp -o main   # DEBUG определён
g++ main.cpp -o main           # DEBUG не определён

# проверка в коде:
#ifdef DEBUG
    std::cout << "debug режим\n";
#endif
```

## `#if` и `defined()`

```cpp
#include <iostream>

#define VERSION 2

#if VERSION >= 2
    void newFeature() { std::cout << "новая фича\n"; }
#else
    void newFeature() { std::cout << "старая фича\n"; }
#endif

// defined() — проверка что макрос определён, без учёта значения
#if defined(DEBUG) && !defined(RELEASE)
    #define LOG(x) std::cout << x << "\n"
#else
    #define LOG(x)
#endif

int main() {
    newFeature();
    LOG("тест");
}
```

`#if` работает только с целочисленными константными выражениями — нельзя сравнивать строки.

## Определение компилятора и компилятороспецифичные `#pragma`

```cpp
#include <iostream>

int main() {
#if defined(__GNUC__) && !defined(__clang__)
    std::cout << "GCC\n";
#elif defined(__clang__)
    std::cout << "Clang\n";
#elif defined(_MSC_VER)
    std::cout << "MSVC\n";
#endif
}
```

`__GNUC__` определён и в GCC, и в Clang — проверять Clang нужно первым или дополнительным `!defined(__clang__)`.

### `#pragma` — игнорирование предупреждений (GCC/Clang)

```cpp
#include <iostream>

void foo() {
#pragma GCC diagnostic push
#pragma GCC diagnostic ignored "-Wunused-variable"
    int unused_var = 42;  // предупреждение подавлено
#pragma GCC diagnostic pop
    // здесь предупреждение снова активно
}

int main() { foo(); }
```

`push`/`pop` — сохраняем и восстанавливаем состояние предупреждений, подавление действует только на конкретный кусок кода.

MSVC использует другой синтаксис:

```cpp
#pragma warning(push)
#pragma warning(disable: 4101)
// код
#pragma warning(pop)
```

Кроссплатформенно — через проверку компилятора:

```cpp
#if defined(__GNUC__) || defined(__clang__)
    #pragma GCC diagnostic push
    #pragma GCC diagnostic ignored "-Wunused-variable"
#elif defined(_MSC_VER)
    #pragma warning(push)
    #pragma warning(disable: 4101)
#endif

int unused = 1;

#if defined(__GNUC__) || defined(__clang__)
    #pragma GCC diagnostic pop
#elif defined(_MSC_VER)
    #pragma warning(pop)
#endif
```

## Проблемы с аргументами макросов

### Приоритет операторов

Макрос — текстовая подстановка, без скобок результат непредсказуем:

```cpp
#include <iostream>

#define SQUARE_BAD(x) x * x  // плохо — нет скобок

int main() {
    std::cout << SQUARE_BAD(2 + 3) << "\n";
    // подставится: 2 + 3 * 2 + 3 = 2 + 6 + 3 = 11, а не 25!
}
```

**Решение — скобки вокруг каждого аргумента и всего выражения:**

```cpp
#define SQUARE(x) ((x) * (x))

int main() {
    std::cout << SQUARE(2 + 3) << "\n";
    // ((2 + 3) * (2 + 3)) = 25 — правильно!
}
```

### Повторное вычисление

Аргумент подставляется везде где встречается в теле — побочные эффекты выполняются несколько раз:

```cpp
#include <iostream>

#define SQUARE(x) ((x) * (x))

int counter = 0;
int increment() { return ++counter; }

int main() {
    std::cout << SQUARE(increment()) << "\n";
    // ((increment()) * (increment())) — вызовется ДВАЖДЫ: 1 * 2 = 2, не 1!
    std::cout << counter << "\n";  // 2
}
```

**Решение — `inline`-функция вместо макроса:**

```cpp
inline int square(int x) { return x * x; }  // x вычисляется один раз

int main() {
    std::cout << square(increment()) << "\n";  // increment() вызовется один раз
}
```

`inline` — подсказка компилятору подставить тело функции прямо в место вызова вместо реального вызова. Также разрешает множественные определения функции в разных `.cpp` при подключении одного заголовка:

```cpp
// header.h
inline int square(int x) { return x * x; }  // ok — без inline была бы ошибка линковки
```

## Проблемы с несколькими statement и `do {} while(0)`

### Проблема

```cpp
#include <iostream>

#define LOG(msg) { std::cout << "[LOG] " << msg << "\n"; counter++; }

int counter = 0;

int main() {
    bool condition = true;

    if (condition)
        LOG("hello");  // подставится { ... }; — лишняя точка с запятой!
    else
        std::cout << "else\n";
}
```

После подстановки:

```cpp
if (condition)
    { std::cout << "[LOG] " << "hello" << "\n"; counter++; };  // ; после } — лишний statement
else                                                            // else "висит в воздухе"
    std::cout << "else\n";
```

```
error: 'else' without a previous 'if'
```

### Решение — `do { } while(0)`

```cpp
#define LOG(msg) do { std::cout << "[LOG] " << msg << "\n"; counter++; } while(0)

int main() {
    bool condition = true;

    if (condition)
        LOG("hello");
    else
        std::cout << "else\n";
}
```

После подстановки:

```cpp
if (condition)
    do { std::cout << "[LOG] " << "hello" << "\n"; counter++; } while(0);  // ОДИН statement!
else
    std::cout << "else\n";
```

`do {} while(0)` выполняется ровно один раз и требует `;` после себя как обычный вызов функции — `if/else` видит его как единый блок. Просто убрать одну `;` ненадёжно: пользователь по привычке всё равно почти всегда поставит `;` сам, и проблема вернётся.

## Проблемы с запятыми внутри аргументов макроса

Препроцессор **не понимает C++ синтаксис** — для него группирующие скобки это только круглые `()`. Запятая внутри `{}`, `<>` или просто в выражении — разделитель аргументов:

```cpp
#define MY_ASSERT(cond) /* ... */

MY_ASSERT(v == std::vector<int>{1, 2, 3});
// error: macro "MY_ASSERT" passed 3 arguments, but takes just 1
// препроцессор режет по запятым в <int> и в {1, 2, 3}

MY_ASSERT({1, 2, 3});
// error: 3 arguments — фигурные скобки НЕ защищают от разрезания!

MY_ASSERT(foo(map<int, int>{}));
// error: 2 arguments — запятая в <int, int> ломает счёт
```

### Решение — обернуть весь аргумент в круглые скобки

```cpp
MY_ASSERT((v == std::vector<int>{1, 2, 3}));     // 1 аргумент — скобки спасли
MY_ASSERT(({1, 2, 3}));                          // 1 аргумент
MY_ASSERT((foo(map<int, int>{})));               // 1 аргумент
```

Круглые скобки "прячут" внутренние запятые от препроцессора — для него весь `(...)` становится одним токеном.

**Общее правило:** любая незащищённая круглыми скобками запятая на верхнем уровне аргумента — режет счёт аргументов макроса, без исключений для `{}` и `<>`.

## Вариадические макросы

Макрос с переменным числом аргументов — через `...` и `__VA_ARGS__`:

```cpp
#include <cstdio>

#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)

int main() {
    LOG("x = %d, y = %d\n", 10, 20);
    // printf("x = %d, y = %d\n", 10, 20)
}
```

### Проблема — лишняя запятая без аргументов

```cpp
#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)

int main() {
    LOG("просто сообщение\n");
    // printf("просто сообщение\n", )  ← лишняя запятая перед )!
}
```

```
error: expected expression
```

### Решение — `##__VA_ARGS__` (расширение GCC/Clang)

```cpp
#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)

int main() {
    LOG("просто сообщение\n");  // printf("просто сообщение\n") — запятая удалена!
    LOG("x = %d\n", 10);        // printf("x = %d\n", 10) — запятая на месте
}
```

`##` перед `__VA_ARGS__` — если он пуст, предшествующая запятая удаляется автоматически.

### Решение — `__VA_OPT__` (стандартный, C++20)

```cpp
#define LOG(fmt, ...) printf(fmt __VA_OPT__(,) __VA_ARGS__)

int main() {
    LOG("просто сообщение\n");  // printf("просто сообщение\n")
    LOG("x = %d\n", 10);        // printf("x = %d\n", 10)
}
```

`__VA_OPT__(,)` вставляет запятую только если есть хотя бы один вариадический аргумент — стандартное решение, не зависит от компилятора.
