# 62. Трюки с наследованием классов

## `private` / `protected` / `public` наследование

```cpp
struct A {
    int x = 42;
    void foo() {}
};

struct B : public A {};     // public — всё как в A
struct C : protected A {};  // protected — public становится protected
struct D : private A {};    // private — всё становится private
```

| Член в A | `public` наследование | `protected` наследование | `private` наследование |
|---|---|---|---|
| `public` | `public` | `protected` | `private` |
| `protected` | `protected` | `protected` | `private` |
| `private` | недоступно | недоступно | недоступно |

```cpp
#include <iostream>

struct D : private A {
    void bar() { foo(); x = 1; }  // внутри D — доступно (унаследованное)
    int y = 42;                    // своё поле — public по умолчанию (struct)
};

struct E : D {
    void baz() { /* foo(); */ }   // ошибка! A приватна в D — E не видит
};

int main() {
    D d;
    d.y;       // ok — своё поле D
    // d.foo(); // ошибка — унаследованное стало private
}
```

### `struct` vs `class`

Единственное отличие — доступ по умолчанию:

```cpp
struct A { int x; };  // x — public по умолчанию
class  B { int x; };  // x — private по умолчанию

struct C : A {};  // public наследование по умолчанию
class  D : A {};  // private наследование по умолчанию
```

## Slicing (срезка объектов)

Срезка при присваивании производного объекта в переменную базового типа **по значению**:

```cpp
#include <iostream>
#include <string>

struct Animal {
    virtual std::string sound() { return "..."; }
};

struct Dog : Animal {
    std::string name = "Buddy";
    std::string sound() override { return "Woof"; }
};

int main() {
    Dog dog;
    Animal a = dog;                    // срезка! name потерян
    std::cout << a.sound() << "\n";    // "..." — не "Woof"!
}
```

### Защита — ссылки или умные указатели

```cpp
int main() {
    Dog dog;

    Animal& ref = dog;                  // ссылка — срезки нет
    std::cout << ref.sound() << "\n";   // "Woof"

    auto ptr = std::make_unique<Dog>();
    Animal* pa = ptr.get();             // указатель — срезки нет
    std::cout << pa->sound() << "\n";   // "Woof"
}
```

### Защита — запретить копирование базового класса

```cpp
struct Animal {
    Animal() = default;
    Animal(const Animal&) = delete;
    Animal& operator=(const Animal&) = delete;

    virtual std::string sound() { return "..."; }
};

int main() {
    Dog dog;
    // Animal a = dog;  // ошибка компиляции — копирование запрещено
}
```

## Получение most derived class через `dynamic_cast`

**Most derived class** — реальный тип объекта, которым он был создан. Через указатель на базу видна только часть объекта, но реальный тип не меняется:

```cpp
#include <iostream>

struct A { virtual ~A() = default; };
struct B : A {};
struct C : B {};

int main() {
    A* pa = new C();  // most derived — C

    C* pc = dynamic_cast<C*>(pa);  // ok — реальный тип C
    B* pb = dynamic_cast<B*>(pa);  // ok — C наследует B

    struct D : A {};
    D* pd = dynamic_cast<D*>(pa);  // nullptr — C не связан с D

    if (pc) std::cout << "это C!\n";

    // указатель на начало most derived объекта
    void* most_derived = dynamic_cast<void*>(pa);
}
```

`dynamic_cast` читает `type_info` из vtable — там записан реальный тип. Затем проверяет есть ли путь от реального типа к запрошенному: если да — возвращает указатель, если нет — `nullptr`.

## Делегирующие конструкторы

Один конструктор вызывает другой конструктор того же класса — избегаем дублирования логики:

```cpp
#include <iostream>
#include <string>

struct Foo {
    int x;
    std::string s;

    Foo(int x, std::string s) : x(x), s(s) {
        std::cout << "основной конструктор\n";
    }

    Foo() : Foo(0, "default") {          // делегирует основному
        std::cout << "делегирующий\n";
    }

    Foo(int x) : Foo(x, "no string") {}  // делегирует с другими аргументами
};

int main() {
    Foo f1;              // основной → делегирующий
    Foo f2(42);           // основной
    Foo f3(1, "hello");   // основной
}
```

### Поведение при исключениях

Если делегируемый конструктор завершился успешно — объект считается **созданным**. Если тело делегирующего бросит — деструктор **вызовется**:

```cpp
#include <stdexcept>

struct Bar {
    Bar(int x) { std::cout << "основной\n"; }  // успешно — объект создан

    Bar() : Bar(42) {                      // делегирует — успешно
        throw std::runtime_error("упс");   // бросает после делегирования
    }

    ~Bar() { std::cout << "~Bar\n"; }      // вызовется! объект был создан
};

int main() {
    try { Bar b; }
    catch (...) { std::cout << "поймали\n"; }
    // основной → ~Bar → поймали
}
```

## Таблица виртуальных функций (vtable) с наследованием

### Без наследования

```cpp
#include <iostream>

struct A {
    virtual void foo() { std::cout << "A::foo\n"; }
    virtual void bar() { std::cout << "A::bar\n"; }
    int x = 1;
};
```

```
A в памяти:        vtable_A:
[ vptr | x ]   →   [ A::foo ]
                   [ A::bar ]
```

### С наследованием — компилятор копирует и заменяет

```cpp
struct B : A {
    void foo() override { std::cout << "B::foo\n"; }  // переопределяет
    int y = 2;
};
```

```
vtable_A:        vtable_B:
[ A::foo ]  →   [ B::foo ]  ← заменили
[ A::bar ]  →   [ A::bar ]  ← оставили без изменений
```

```cpp
int main() {
    B obj;
    A* pa = &obj;
    pa->foo();  // vptr → vtable_B → B::foo
    pa->bar();  // vptr → vtable_B → A::bar
}
```

### Цепочка наследования

```cpp
struct C : B {
    void bar() override { std::cout << "C::bar\n"; }
};
```

```
vtable_A:    vtable_B:    vtable_C:
[ A::foo ]   [ B::foo ]   [ B::foo ]  ← унаследовано от B
[ A::bar ]   [ A::bar ]   [ C::bar ]  ← заменили
```

```cpp
int main() {
    C obj;
    A* pa = &obj;
    pa->foo();  // B::foo
    pa->bar();  // C::bar
}
```

### Накладные расходы

```cpp
struct WithVirtual { virtual void foo() {} };
struct WithoutVirtual { void foo() {} };

int main() {
    std::cout << sizeof(WithVirtual)    << "\n";  // 8 — только vptr
    std::cout << sizeof(WithoutVirtual) << "\n";  // 1 — пустой класс
}
```

- каждый объект хранит `vptr` (8 байт на 64-bit)
- вызов виртуальной функции — косвенный, через таблицу
- компилятор обычно не может инлайнить виртуальные вызовы

## CRTP (Curiously Recurring Template Pattern)

Класс наследует от шаблона **самого себя**:

```cpp
#include <iostream>

template<typename Derived>
struct Base {
    void foo() {
        static_cast<Derived*>(this)->impl();  // даункаст к наследнику
    }
};

struct MyClass : Base<MyClass> {  // передаём себя как параметр
    void impl() { std::cout << "MyClass::impl\n"; }
};

int main() {
    MyClass obj;
    obj.foo();  // MyClass::impl
}
```

`static_cast<Derived*>(this)` легален потому что `MyClass` **гарантированно наследует** `Base<MyClass>` — обычный downcast по простой (не виртуальной) иерархии. Это соглашение CRTP: программист обязан написать именно `struct MyClass : Base<MyClass>`.

### Зачем — переиспользуемая логика поверх чужого метода

```cpp
#include <iostream>

template<typename Derived>
struct Printable {
    void print_twice() {           // логика общая для всех наследников
        auto* self = static_cast<Derived*>(this);
        self->print();
        self->print();
    }
};

struct Greeting : Printable<Greeting> {
    void print() { std::cout << "Hello!\n"; }
};

struct Farewell : Printable<Farewell> {
    void print() { std::cout << "Bye!\n"; }
};

int main() {
    Greeting g;
    g.print_twice();  // Hello! Hello! — получили бесплатно

    Farewell f;
    f.print_twice();  // Bye! Bye! — та же логика, другой print()
}
```

Каждый класс пишет только простой метод (`print`), а сложную общую логику (`print_twice`) пишем один раз в `Base` — без дублирования и без vtable.

### Зачем — статический полиморфизм

```cpp
#include <iostream>

template<typename Derived>
struct Animal {
    void describe() {
        static_cast<Derived*>(this)->sound();  // инлайнится — быстро
    }
};

struct Dog : Animal<Dog> {
    void sound() { std::cout << "Woof\n"; }
};

struct Cat : Animal<Cat> {
    void sound() { std::cout << "Meow\n"; }
};

int main() {
    Dog dog; dog.describe();  // Woof
    Cat cat; cat.describe();  // Meow
}
```

### CRTP vs виртуальные функции

| | Виртуальные функции | CRTP |
|---|---|---|
| Полиморфизм | рантайм | компайл-тайм |
| Скорость | медленнее (vtable) | быстро (инлайн) |
| Хранить в одном контейнере | можно (`vector<Base*>`) | нельзя — разные типы |
| Гибкость | тип можно менять в рантайме | тип фиксирован на компиляции |

Главный минус CRTP — `Animal<Dog>` и `Animal<Cat>` это **разные типы**, нельзя сложить их в один `std::vector` как с виртуальными функциями.
