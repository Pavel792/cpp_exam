# 60. Множественное наследование

## Синтаксис

```cpp
#include <iostream>

struct A {
    int a = 1;
    void foo() { std::cout << "A::foo\n"; }
};

struct B {
    int b = 2;
    void bar() { std::cout << "B::bar\n"; }
};

struct C : A, B {  // наследуем от A и B
    int c = 3;
};

int main() {
    C obj;
    obj.foo();  // из A
    obj.bar();  // из B
    std::cout << obj.a << " " << obj.b << " " << obj.c << "\n";
}
```

---

## Представление в памяти

Подобъекты лежат последовательно в порядке объявления:

```
C в памяти:
[ A подобъект | B подобъект | поля C ]
[ a=1         | b=2         | c=3    ]
  ↑             ↑
  адрес C       адрес C + sizeof(A)
```

### Изменение адреса при `static_cast`

При приведении к `B*` адрес **смещается** — компилятор добавляет sizeof(A):

```cpp
#include <iostream>

struct A { int a = 1; };  // 4 байта
struct B { int b = 2; };  // 4 байта
struct C : A, B { int c = 3; };

int main() {
    C obj;
    C* pc = &obj;
    A* pa = static_cast<A*>(pc);  // смещение 0 — A первый подобъект
    B* pb = static_cast<B*>(pc);  // смещение +4 — B после A

    std::cout << (void*)pc << "\n";  // 0x...00
    std::cout << (void*)pa << "\n";  // 0x...00 — то же
    std::cout << (void*)pb << "\n";  // 0x...04 — сдвинут!

    // обратно — static_cast вычитает смещение
    C* pc2 = static_cast<C*>(pb);
    std::cout << (void*)pc2 << "\n";  // 0x...00 — вернулись
}
```

**Никогда не используй `reinterpret_cast` между базой и наследником** — он не добавляет смещение:

```cpp
B* pb_bad = reinterpret_cast<B*>(pc);  // адрес не сдвинут — UB!
```

## Порядок инициализации и уничтожения

1. Виртуальные базы — самым производным классом
2. Невиртуальные базы — в порядке объявления
3. Поля — в порядке объявления
4. Тело конструктора

Уничтожение — строго обратный порядок.

```cpp
#include <iostream>

struct A {
    A() { std::cout << "A()\n"; }
    ~A() { std::cout << "~A()\n"; }
};

struct B {
    B() { std::cout << "B()\n"; }
    ~B() { std::cout << "~B()\n"; }
};

struct C : A, B {
    C() : A(), B() {        // порядок в списке не важен —
        std::cout << "C()\n"; // важен порядок объявления: A, B
    }
    ~C() { std::cout << "~C()\n"; }
};

int main() {
    C obj;
    // A() → B() → C() → ~C() → ~B() → ~A()
}
```

### Передача параметров конструкторам баз

```cpp
struct A { A(int x) { std::cout << "A(" << x << ")\n"; } };
struct B { B(std::string s) { std::cout << "B(" << s << ")\n"; } };

struct C : A, B {
    C(int x, std::string s) : A(x), B(s) {}  // передаём явно
};

int main() {
    C obj(42, "hello");  // A(42) → B(hello)
}
```

## Дублирование базового класса — ромбовидное наследование

```
    A
   / \
  B   C
   \ /
    D
```

```cpp
#include <iostream>

struct A { int x = 42; };
struct B : A {};
struct C : A {};
struct D : B, C {};  // два подобъекта A!

int main() {
    D obj;
    // obj.x;          // ошибка — неоднозначность, какой A?
    obj.B::x = 1;      // явно через B
    obj.C::x = 2;      // явно через C — независимые копии!

    // A* pa = &obj;   // ошибка — какой A брать?
    A* pa = static_cast<B*>(&obj);  // явно через B
}
```

### Решение — виртуальное наследование

```cpp
#include <iostream>

struct A {
    int x = 42;
    A(int x) : x(x) { std::cout << "A(" << x << ")\n"; }
};

struct B : virtual A {
    B() : A(1) {}  // игнорируется при создании D!
};

struct C : virtual A {
    C() : A(2) {}  // игнорируется при создании D!
};

struct D : B, C {
    D() : A(42), B(), C() {}  // D сам инициализирует A — обязательно!
};

int main() {
    D obj;        // A(42) — один экземпляр
    obj.x;        // ok — неоднозначности нет
    A* pa = &obj; // ok — один A
}
```

В памяти:
```
без виртуального:
D: [ B[ копия A ] | C[ копия A ] | поля D ]  ← два A

с виртуальным:
D: [ B[vptr|поля B] | C[vptr|поля C] | поля D | A ]  ← один A
        ↓                  ↓
   таблица смещений   таблица смещений
   до общего A        до общего A
```

## Cross-cast (Side-cast)

Приведение **поперёк** иерархии — между независимыми базами. Только `dynamic_cast`:

```cpp
#include <iostream>

struct A { virtual ~A() = default; };
struct B : virtual A {};
struct C : virtual A {};
struct D : B, C {};

int main() {
    D obj;
    B* pb = &obj;

    // static_cast — ошибка компиляции, нет пути B → C
    C* pc = dynamic_cast<C*>(pb);  // ok — идёт через RTTI: B → D → C
    std::cout << (pc ? "успех" : "nullptr") << "\n";  // успех
}
```

`dynamic_cast` при cross-cast: смотрит реальный тип через RTTI → находит путь → вычисляет смещение.

## Касты при множественном наследовании

### Неявное преобразование — только вверх

```cpp
C obj;
A* pa = &obj;  // ok — неявно, смещение 0
B* pb = &obj;  // ok — неявно, адрес сдвигается
// C* pc = pa; // ошибка — вниз только явно
```

### `static_cast` — вверх и вниз, без проверки

```cpp
B* pb = static_cast<B*>(&obj);   // вверх — со смещением
C* pc = static_cast<C*>(pb);     // вниз — вычитает смещение

A* pa = new A();
C* pc2 = static_cast<C*>(pa);    // компилируется, но UB если pa не C!
```

### `dynamic_cast` — вверх, вниз, поперёк

```cpp
A* pa = &obj;
D* pd = dynamic_cast<D*>(pa);   // вниз — проверяет реальный тип
B* pb = dynamic_cast<B*>(pa);   // вниз — ok
C* pc = dynamic_cast<C*>(pb);   // cross-cast — только dynamic_cast

// неудача
struct E : A {};
E* pe = dynamic_cast<E*>(pa);   // nullptr — реальный тип D, не E
```

## `override` при множественном наследовании

Если обе базы имеют одинаковую виртуальную функцию:

```cpp
#include <iostream>

struct A { virtual void foo() { std::cout << "A::foo\n"; } };
struct B { virtual void foo() { std::cout << "B::foo\n"; } };

struct C : A, B {
    void foo() override { std::cout << "C::foo\n"; }
    // override заменяет foo в обеих vtable
};

int main() {
    C obj;
    obj.foo();   // C::foo

    A* pa = &obj;
    pa->foo();   // C::foo — через vtable_A

    B* pb = &obj;
    pb->foo();   // C::foo — через vtable_B + thunk корректирует this
}
```

В памяти две vtable, обе указывают на `C::foo`:

```
C в памяти:
[ vptr_A | поля A | vptr_B | поля B | поля C ]
    ↓                  ↓
  vtable_A           vtable_B
  [C::foo]           [thunk → C::foo]  ← thunk корректирует this
```

**Без `override`** — скрытие (hiding), не переопределение. `override` добавляет проверку компилятора:

```cpp
void fooo() override { }  // ошибка компиляции — опечатка, нет такой в базе
```

## Приватное и защищённое наследование

```cpp
#include <iostream>

struct A {
    int x = 42;
    void foo() { std::cout << "A::foo\n"; }
};

struct B : private A {     // приватное — A скрыта снаружи
    void bar() { foo(); }  // внутри B — доступно
};

struct C : protected A {   // protected — доступно наследникам
    void baz() { foo(); }  // внутри C — доступно
};

struct D : C {
    void qux() { foo(); }  // внутри D — доступно (protected)
};

int main() {
    B b;
    // b.foo();              // ошибка — A приватна
    // A* pa = &b;           // ошибка — неявный upcasting закрыт
    // static_cast<A*>(&b);  // ошибка — static_cast тоже закрыт

    C c;
    // c.foo();              // ошибка — protected, не public
}
```

| | `public` | `protected` | `private` |
|---|---|---|---|
| Внутри класса | ✅ | ✅ | ✅ |
| В наследнике | ✅ | ✅ | ❌ |
| Снаружи | ✅ | ❌ | ❌ |
| Неявный upcasting | ✅ | ❌ | ❌ |
