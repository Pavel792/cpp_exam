# Си

## char s[10] vs const char* s

Две принципиально разные сущности, которые в определённых контекстах выглядят взаимозаменяемо, но фундаментально различаются.

### char s[10]

Массив — блок памяти фиксированного размера (10 байт), выделенный там, где объявлена переменная (стек для локальной переменной). Имя `s` в большинстве выражений распадается (decay) в указатель на первый элемент, но сам `s` — не указатель, а массив:

```cpp
char s[10] = "hi"; // массив 10 байт: 'h', 'i', '\0', и 7 неинициализированных байт

sizeof(s); // 10 — размер ВСЕГО массива
```

Память под `s` существует ровно столько, сколько живёт сама переменная — массив владеет этой памятью.

### const char* s

Указатель — переменная, хранящая адрес в памяти, где лежат символы. Сам указатель занимает фиксированный размер (8 байт на x86_64), независимо от длины строки:

```cpp
const char* s = "hi"; // s хранит адрес строкового литерала "hi" в read-only памяти

sizeof(s); // 8 — размер указателя, а не строки
```

Указатель не владеет памятью, на которую указывает. Строковые литералы обычно `const`, поэтому используется `const char*` — попытка изменить строковый литерал через `char*` без `const` — UB:

```cpp
char s[10] = "hi";
s[0] = 'H'; // OK — массив, можно изменять содержимое

const char* p = "hi";
p[0] = 'H'; // UB — попытка изменить строковый литерал (обычно read-only сегмент)
```

### Ключевое практическое отличие

```cpp
char s[10];
s = "world"; // ошибка компиляции — массив нельзя переприсвоить целиком

const char* p;
p = "world"; // OK — указатель можно перенаправить на другую строку
```

`s` как массив фиксирован раз и навсегда по адресу и размеру; `p` как указатель можно свободно переназначать.

## Преобразование в обе стороны для std::string

### C-style → std::string

```cpp
const char* cstr = "hello";
std::string s(cstr);    // конструктор std::string принимает const char*
std::string s2 = cstr;  // эквивалентно — неявная конвертация через конструктор

char arr[10] = "hi";
std::string s3(arr);     // массив decay'ится в const char*, дальше как обычно
```

`std::string` копирует символы из C-строки в собственный внутренний буфер — после этого полностью независим от исходного `cstr`/`arr`.

### std::string → C-style

```cpp
std::string s = "hello";
const char* cstr = s.c_str(); // указатель на внутренний буфер std::string, гарантированно null-terminated
const char* data = s.data();  // с C++11 эквивалентно c_str() для const-доступа
```

`c_str()`/`data()` возвращают указатель на внутреннюю память самого `std::string`, а не копию. Этот указатель действителен только пока жив исходный `std::string`, и инвалидируется при любой операции, способной вызвать реаллокацию или изменение строки:

```cpp
const char* dangerous;
{
    std::string temp = "hello";
    dangerous = temp.c_str(); // указывает на буфер temp
} // temp уничтожен — dangerous теперь dangling pointer

std::cout << dangerous; // UB
```

## Конкатенация, получение длины, безопасные альтернативы

### Конкатенация

C-строки не поддерживают `+` — конкатенация через `strcat` из `<cstring>`, дописывающую вторую строку в конец первой, требуя достаточно места в буфере первой:

```cpp
char buf[20] = "Hello, ";
strcat(buf, "World!"); // buf теперь "Hello, World!"
// опасно: если в buf не хватит места — UB (переполнение буфера)
```

Более безопасная версия — `strncat`, ограничивающая количество копируемых символов:

```cpp
strncat(buf, "World!", sizeof(buf) - strlen(buf) - 1);
```

### Получение длины

`strlen` возвращает количество символов до первого `'\0'` (не включая его), за O(n), так как C-строки не хранят длину отдельно:

```cpp
const char* s = "hello";
strlen(s); // 5

char buf[20] = "hi";
strlen(buf); // 2 — длина содержимого, а не sizeof(buf) (которое было бы 20)
```

Важно различать `strlen(s)` (длина строки до `\0`) и `sizeof(s)` (размер буфера или указателя).

### Безопасные альтернативы gets и scanf("%s")

`gets()` не имеет способа ограничить количество читаемых символов, поэтому считается настолько опасной (классический источник переполнения буфера), что полностью удалена из стандарта C++ (с C++14) и C11:

```cpp
char buf[10];
gets(buf); // удалена из стандарта — никогда не используется
            // если ввод длиннее 9 символов — переполнение буфера
```

Безопасная альтернатива — `fgets`, принимающая максимальный размер буфера:

```cpp
char buf[10];
fgets(buf, sizeof(buf), stdin); // читает максимум 9 символов + '\0'
```

`scanf("%s", buf)` имеет ту же проблему — не ограничивает количество считываемых символов по умолчанию:

```cpp
char buf[10];
scanf("%s", buf); // опасно — если ввод длиннее 9 символов, переполнение буфера
```

Безопасная альтернатива — указать максимальную ширину поля в формат-строке:

```cpp
char buf[10];
scanf("%9s", buf); // читает максимум 9 символов + '\0' — безопасно
```

Для C++ универсальная рекомендация — использовать `std::cin >> std::string` или `std::getline`, которые сами управляют памятью без проблемы переполнения буфера:

```cpp
std::string s;
std::cin >> s;             // безопасно, std::string сам управляет памятью
std::getline(std::cin, s); // безопасно, читает всю строку целиком, включая пробелы
```

## Массивы массивов (int**): выделение, освобождение, использование

`int**` — указатель на указатель, часто используется для реализации "двумерного массива" как массива указателей на отдельно выделенные строки (в отличие от настоящего непрерывного 2D-массива).

### Выделение

```cpp
int rows = 3, cols = 4;

int** matrix = new int*[rows]; // массив из 3 указателей на int (каждый — будущая строка)

for (int i = 0; i < rows; i++) {
    matrix[i] = new int[cols]; // выделяем память под каждую строку отдельно
}
```

`matrix` указывает на массив из `rows` указателей, каждый из которых указывает на отдельно выделенный блок из `cols` элементов `int`. Эти блоки не обязательно расположены подряд в памяти — каждая строка выделена отдельным вызовом `new[]` и может оказаться где угодно в куче.

### Использование

```cpp
matrix[1][2] = 42; // эквивалентно *(*(matrix + 1) + 2) = 42
// matrix[1] -> указатель на вторую строку (int*)
// matrix[1][2] -> третий элемент этой строки

for (int i = 0; i < rows; i++) {
    for (int j = 0; j < cols; j++) {
        matrix[i][j] = i * cols + j;
    }
}
```

### Освобождение

Должно происходить в порядке, обратном выделению: сначала каждая строка отдельно, потом сам массив указателей:

```cpp
for (int i = 0; i < rows; i++) {
    delete[] matrix[i]; // освобождаем каждую строку
}
delete[] matrix; // освобождаем массив указателей
```

Если забыть освободить отдельные строки и сразу сделать `delete[] matrix`, память строк утечёт — `delete[] matrix` освобождает только массив указателей, но не данные, на которые они указывали:

```cpp
delete[] matrix; // ошибка: строки matrix[0], matrix[1], matrix[2] никогда не освобождены — утечка
```

В реальном C++-коде такую конструкцию почти никогда не пишут вручную, предпочитая `std::vector<std::vector<int>>` (сам управляет памятью корректно по правилу нуля) или, для производительности, один непрерывный `std::vector<int>` размера `rows * cols` с ручным пересчётом индексов (`v[i * cols + j]`) — это даёт настоящую непрерывность памяти и кэш-эффективность, которой не хватает массиву массивов.
