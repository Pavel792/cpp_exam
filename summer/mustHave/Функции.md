# Функции

## Объявление и определение функции

Объявление (declaration) сообщает компилятору о существовании функции — имя, тип возвращаемого значения, типы параметров — но не содержит тела:

```cpp
int add(int a, int b); // объявление ("прототип")
```

Определение (definition) включает тело функции — реальную реализацию:

```cpp
int add(int a, int b) { // определение
    return a + b;
}
```

Объявление можно делать сколько угодно раз (если они идентичны друг другу), а определение функции с данной сигнатурой в программе должно быть ровно одно — правило ODR (One Definition Rule). Объявление без тела нужно компилятору, чтобы знать о существовании функции до того, как он увидит её полное определение:

```cpp
// header.h
int add(int a, int b); // объявление — видно всем, кто подключит заголовок

// main.cpp
#include "header.h"
int main() {
    add(2, 3); // вызов на основе объявления
}

// add.cpp
#include "header.h"
int add(int a, int b) { return a + b; } // определение
```

Определение функции одновременно является и её объявлением (отдельное объявление не требуется, если определение идёт раньше места использования в этом же файле). Но не наоборот: объявление без тела не заменяет определение. Если в программе нигде нет определения для объявленной и вызванной функции — ошибка линковки (linker error), а не компиляции, потому что каждая единица трансляции (`.cpp`-файл) удовлетворена одним объявлением, а линковщик уже не может найти реализацию при сборке финального исполняемого файла:

```cpp
void foo(); // объявление

int main() {
    foo(); // компилируется нормально — объявление есть
}
// ошибка ЛИНКОВКИ: undefined reference to `foo()` — определения нигде нет
```

## Вызов функции

Синтаксис вызова — имя функции, за ним список аргументов в скобках:

```cpp
int result = add(2, 3); // вызов функции add с аргументами 2 и 3
```

Количество и типы переданных аргументов должны соответствовать объявленным параметрам, с учётом неявных преобразований типов и возможных значений по умолчанию:

```cpp
int multiply(int a, int b = 2); // b имеет значение по умолчанию

multiply(5);     // эквивалентно multiply(5, 2)
multiply(5, 3);  // explicit — multiply(5, 3)
```

На уровне исходного кода вызов — это применение оператора `()` к имени функции с аргументами; под капотом компилятор размещает аргументы согласно ABI платформы, передаёт управление по адресу функции и возвращается обратно, но эти детали (конвенции вызова, mangling) не входят в базовый ответ.

## Возвращаемое значение

Тип возвращаемого значения указывается перед именем функции. Оператор `return` завершает выполнение функции и передаёт значение обратно в точку вызова:

```cpp
int square(int x) {
    return x * x;
}
```

Если функция ничего не возвращает, используется тип `void`:

```cpp
void printHello() {
    std::cout << "Hello";
    // явный return не обязателен в конце void-функции
}
```

Если функция объявлена с возвращаемым типом, отличным от `void`, и не выполняет `return` на каком-то из путей выполнения, это UB при достижении конца функции:

```cpp
int f(bool flag) {
    if (flag) {
        return 1;
    }
    // если flag == false — нет return, UB при достижении конца функции
}
```

`return` может выполниться в любой точке функции, не обязательно в конце — выполнение немедленно прекращается в момент `return`:

```cpp
int abs(int x) {
    if (x < 0) {
        return -x; // выход здесь, остальной код не выполняется
    }
    return x;
}
```

Возвращаемое значение копируется (или перемещается) в точку вызова. Если функция возвращает локальную переменную по значению, это безопасно (в отличие от возврата ссылки на неё), потому что компилятор делает копию значения до уничтожения локальной переменной — а зачастую, благодаря copy elision (NRVO/RVO), копии вообще не происходит, объект сразу строится в месте назначения.

## Рекурсивный вызов

Рекурсия — ситуация, когда функция вызывает саму себя, прямо или через цепочку других функций:

```cpp
int factorial(int n) {
    if (n <= 1) {
        return 1; // базовый случай — останавливает рекурсию
    }
    return n * factorial(n - 1); // рекурсивный вызов
}
```

Два обязательных компонента корректной рекурсии: базовый случай (условие, при котором функция возвращает значение без дальнейшего рекурсивного вызова) и рекурсивный случай (вызов функции с "уменьшенными" аргументами, приближающими к базовому случаю). Без базового случая или при ошибке в условии выхода рекурсия становится бесконечной.

Каждый рекурсивный вызов создаёт новый кадр стека (stack frame) со своими локальными переменными и параметрами, независимыми от предыдущих вызовов:

```text
factorial(3)
  -> n=3, вызывает factorial(2)
       -> n=2, вызывает factorial(1)
            -> n=1, базовый случай, return 1
       -> возвращается в n=2: return 2 * 1 = 2
  -> возвращается в n=3: return 3 * 2 = 6
```

Каждый вызов потребляет место в стеке вызовов (call stack), размер которого ограничен (обычно несколько мегабайт по умолчанию). Слишком глубокая рекурсия (отсутствие базового случая или очень большое n) приводит к stack overflow — переполнению стека, что обычно завершает программу аварийно. Формально стандарт C++ тоже не гарантирует поведения при исчерпании ресурсов, но это принципиально иная история, чем UB от переполнения signed-арифметики — здесь дело в ограничении конкретной платформы и ОС.
