# Классы

## Определение класса, конструктор, деструктор, методы

Класс объединяет данные (поля) и поведение (методы) в единую сущность:

```cpp
class Point {
public:
    Point(double x, double y) : x_(x), y_(y) {} // конструктор
    ~Point() {}                                   // деструктор

    double length() const { // метод
        return std::sqrt(x_ * x_ + y_ * y_);
    }

private:
    double x_, y_; // поля
};
```

Конструктор — специальный метод, вызываемый автоматически при создании объекта. Имя совпадает с именем класса, нет возвращаемого типа (даже `void`). Список инициализации (`: x_(x), y_(y)`) предпочтительнее присваивания в теле, поскольку именно инициализирует поля (вызывает конструктор поля напрямую), а не сначала создаёт их значениями по умолчанию, а потом присваивает:

```cpp
Point(double x, double y) : x_(x), y_(y) {} // инициализация — один шаг

Point(double x, double y) {
    x_ = x; // присваивание — сначала поле создано значением по умолчанию, потом присвоено
    y_ = y;
}
```

Для ссылок и `const`-полей список инициализации обязателен — присвоить их после создания нельзя:

```cpp
class Foo {
    const int id_;
public:
    Foo(int id) : id_(id) {} // обязательно — const-поле нельзя присвоить в теле
};
```

Деструктор — специальный метод, вызываемый автоматически при уничтожении объекта (выход из области видимости для автоматических объектов, `delete` для динамических). Имя — `~` плюс имя класса, без параметров, без возвращаемого типа, не может быть перегружен. Отвечает за освобождение ресурсов, которыми владеет объект.

Методы — функции, объявленные внутри класса, с неявным доступом к полям объекта через `this`.

## Специальные методы (пять штук)

Компилятор может автоматически сгенерировать пять специальных методов, если они не объявлены явно:

```cpp
class Foo {
public:
    Foo();                            // 1. конструктор по умолчанию
    Foo(const Foo& other);            // 2. конструктор копирования
    Foo& operator=(const Foo& other); // 3. оператор копирующего присваивания
    Foo(Foo&& other);                 // 4. конструктор перемещения
    Foo& operator=(Foo&& other);      // 5. оператор перемещающего присваивания
    ~Foo();                           // деструктор (иногда считают шестым в списке)
};
```

Конструктор по умолчанию вызывается без аргументов, инициализирует поля их собственными конструкторами по умолчанию.

Конструктор копирования создаёт новый объект как копию существующего:

```cpp
Foo a;
Foo b(a); // конструктор копирования — b независимая копия a
```

Оператор копирующего присваивания копирует состояние одного существующего объекта в другой существующий:

```cpp
Foo a, b;
b = a; // operator= — состояние a копируется в уже существующий b
```

Конструктор перемещения и оператор перемещающего присваивания (C++11+) забирают ресурсы у временного объекта (rvalue) вместо копирования, оставляя исходный объект в валидном, но обычно "пустом" состоянии:

```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b = std::move(a); // b забирает внутренний буфер a
                                     // a в валидном, но "пустом" состоянии
```

Это важно для эффективности: копирование тяжёлого объекта требует выделения новой памяти и копирования всех данных, перемещение — просто передача владения указателем, операция O(1).

## Правило нуля и правило пяти

**Правило пяти.** Если класс явно объявляет хотя бы один из пяти специальных методов (или деструктор), нужно явно объявить и остальные четыре — объявление одного из них обычно сигнализирует, что класс вручную управляет ресурсом (сырой указатель, файловый дескриптор), и автоматически сгенерированные версии остальных методов, скорее всего, окажутся некорректными:

```cpp
class Buffer {
    int* data_;
public:
    Buffer(size_t size) : data_(new int[size]) {}
    ~Buffer() { delete[] data_; } // явный деструктор — управляем ресурсом вручную

    // без явного объявления копирования компилятор сгенерирует побитовое копирование data_:
    // Buffer b2 = b1 -> оба data_ указывают на одну память
    // оба деструктора попытаются delete[] одну и ту же память -> UB (double free)

    Buffer(const Buffer& other) : data_(new int[/* размер */]) {
        // правильное глубокое копирование
    }
    Buffer& operator=(const Buffer& other) { /* ... */ return *this; }
    Buffer(Buffer&& other) noexcept : data_(other.data_) {
        other.data_ = nullptr; // забираем владение, обнуляем источник
    }
    Buffer& operator=(Buffer&& other) noexcept { /* ... */ return *this; }
};
```

**Правило нуля.** Более современная и предпочтительная практика — вообще не объявлять ни один из пяти специальных методов, доверив управление ресурсами специализированным классам-обёрткам (`std::vector`, `std::unique_ptr`, `std::string`), которые уже корректно реализуют правило пяти внутри себя:

```cpp
class Buffer {
    std::vector<int> data_; // vector сам управляет памятью корректно
public:
    Buffer(size_t size) : data_(size) {}
    // явные деструктор/копирование/перемещение не нужны —
    // сгенерированные версии просто делегируют работу в data_
};
```

Правило нуля считается лучшей практикой: меньше ручного кода — меньше шансов на ошибку, а сложность управления ресурсом инкапсулирована один раз внутри `vector`/`unique_ptr`, а не размазана по всем классам.

## Приватные, защищённые и публичные поля и методы

Три уровня доступа управляют видимостью членов класса извне и из производных классов:

```cpp
class Base {
public:
    int publicField;     // доступен отовсюду

protected:
    int protectedField;  // доступен из методов класса и из наследников, но не извне

private:
    int privateField;    // доступен только из методов самого класса
};

class Derived : public Base {
    void foo() {
        publicField = 1;     // OK
        protectedField = 2;  // OK — доступ из наследника
        privateField = 3;    // ошибка — недоступно даже наследнику
    }
};

int main() {
    Base b;
    b.publicField = 1;    // OK
    b.protectedField = 2; // ошибка — недоступно извне
    b.privateField = 3;   // ошибка — недоступно извне
}
```

По умолчанию у `class` все члены `private`, у `struct` — все `public`. Это единственное содержательное различие между `class` и `struct` в C++:

```cpp
class Foo { int x; };   // x — private по умолчанию
struct Bar { int x; };  // x — public по умолчанию
```

## Наследование: базовый синтаксис и вызов конструктора базового класса

Синтаксис — двоеточие и спецификатор доступа перед именем базового класса:

```cpp
class Animal {
public:
    Animal(std::string name) : name_(name) {}
    void speak() const { std::cout << name_ << " makes a sound\n"; }
protected:
    std::string name_;
};

class Dog : public Animal { // Dog наследует от Animal
public:
    Dog(std::string name) : Animal(name) {} // явный вызов конструктора базового класса
    void bark() const { std::cout << name_ << " barks\n"; } // доступ к protected-полю базы
};
```

Конструктор базового класса должен быть вызван до начала выполнения тела конструктора производного класса (база всегда конструируется первой). Это делается через список инициализации производного конструктора:

```cpp
Dog(std::string name) : Animal(name) {} // Animal(name) выполняется ДО тела Dog
```

Если явно не вызвать конструктор базового класса в списке инициализации, компилятор попытается вызвать конструктор базового класса по умолчанию автоматически:

```cpp
class Dog : public Animal {
public:
    Dog(std::string name) {} // Animal не указан явно — компилятор попытается вызвать Animal()
    // ошибка компиляции, если у Animal нет конструктора по умолчанию
};
```

Порядок инициализации при наследовании: сначала конструктор базового класса, затем поля производного класса в порядке их объявления (не в порядке списка инициализации), затем тело конструктора производного класса. Уничтожение — в обратном порядке: сначала тело деструктора производного класса, затем поля производного класса, затем деструктор базового класса.

Тип наследования (`public`, `protected`, `private`) определяет, как изменяются уровни доступа базового класса при взгляде на объект производного класса извне. `public`-наследование (самое частое) сохраняет уровни доступа как есть: `public` базового остаётся `public` у производного, `protected` остаётся `protected`.
