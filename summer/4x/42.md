# 42. Оповещение о событиях

## Условные переменные

Мьютекс защищает данные, но не умеет **ждать события**. Без `condition_variable` пришлось бы крутиться в цикле — **busy waiting**:

```cpp
// плохо — жрёт CPU, мешает другим потокам захватить мьютекс
void car() {
    while (true) {
        std::lock_guard<std::mutex> lock(mtx);
        if (green) break;  // захватили, проверили, отпустили — миллион раз
    }
}
```

`condition_variable` позволяет потоку **реально засыпать** до наступления события:

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
bool green = false;

void car() {
    std::unique_lock<std::mutex> lock(mtx);  // лочит сразу в конструкторе
    cv.wait(lock, []{ return green; });       // спим пока green == false
    std::cout << "машина едет!\n";
}

void traffic_light() {
    std::this_thread::sleep_for(std::chrono::seconds(2));
    {
        std::lock_guard<std::mutex> lock(mtx);
        green = true;  // меняем под мьютексом — защита от гонки
    }
    cv.notify_one();  // будим одну спящую машину
}

int main() {
    std::thread t1(car);
    std::thread t2(traffic_light);
    t1.join();
    t2.join();
}
```

### Как работает `cv.wait(lock, lambda)`

```
1. Вызываем лямбду под мьютексом — green == false
2. Атомарно: отпускаем мьютекс + засыпаем  ← неделимая операция!
3. traffic_light захватывает мьютекс, green = true, отпускает
4. notify_one() — car просыпается, захватывает мьютекс
5. Вызываем лямбду — green == true → выходим из wait
```

Шаг 2 атомарный — иначе `notify` мог бы улететь в пустоту между проверкой и засыпанием.

### `notify_one` vs `notify_all`

```cpp
cv.notify_one();  // будим одну спящую машину
cv.notify_all();  // будим все — но мьютекс захватывают по одному
```

## Spurious wakeup

Поток может проснуться **без `notify`** — особенность реализации на уровне ОС. Поэтому всегда нужно проверять условие:

```cpp
// плохо — проснулись, но green может быть false
cv.wait(lock);

// хорошо — проверяем условие, засыпаем снова если false
cv.wait(lock, []{ return green; });

// эквивалентно:
while (!green) {
    cv.wait(lock);
}
```

Лямбда — это просто callable возвращающий `bool`. `wait` вызывает её и смотрит на результат. Глобальные переменные видны без захвата `[]`, локальные нужно захватывать `[&]`:

```cpp
bool global = false;
cv.wait(lock, []{ return global; });   // глобальная — захват не нужен

void foo() {
    bool local = false;
    cv.wait(lock, [&]{ return local; });  // локальная — захватываем по ссылке
}
```

## Producer-Consumer

Классическая задача — producer производит данные, consumer потребляет. CV идеально подходит: consumer спит пока нет данных.

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>
#include <string>

std::mutex mtx;
std::condition_variable cv;
std::queue<std::string> q;
bool done = false;  // producer закончил работу

void producer() {
    std::vector<std::string> data = {"one", "two", "three", "four"};

    for (const auto& item : data) {
        {
            std::lock_guard<std::mutex> lock(mtx);
            q.push(item);
            std::cout << "произвели: " << item << "\n";
        }
        cv.notify_one();  // будим consumer — есть новый элемент
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }

    {
        std::lock_guard<std::mutex> lock(mtx);
        done = true;  // больше данных не будет
    }
    cv.notify_one();  // будим consumer чтобы увидел done
}

void consumer() {
    while (true) {
        std::unique_lock<std::mutex> lock(mtx);

        // ждём пока очередь не пуста ИЛИ producer не закончил
        // только !q.empty() не хватит — если done и очередь пуста — заснём навсегда
        cv.wait(lock, []{ return !q.empty() || done; });

        if (q.empty() && done) break;  // всё обработали — выходим

        // забираем элемент под мьютексом
        std::string item = q.front();
        q.pop();
        lock.unlock();  // сразу отпускаем — producer может работать параллельно

        std::cout << "потребили: " << item << "\n";
        // следующая итерация — lock захватится снова в начале while
    }
}

int main() {
    std::thread t1(producer);
    std::thread t2(consumer);
    t1.join();
    t2.join();
}
```

### Почему отпускаем мьютекс при обработке

```cpp
lock.unlock();
std::cout << "потребили: " << item << "\n";  // долгая операция
// producer не ждёт — может класть новые элементы параллельно
```

Держать мьютекс во время обработки — лишнее. Producer и consumer работают параллельно.

## `mutable`

`const` метод не может менять поля. `mutable` — исключение: поле можно менять даже в `const` методе.

### Типичный пример — мьютекс в классе

```cpp
#include <iostream>
#include <mutex>
#include <string>

struct ThreadSafeString {
    std::string data;
    mutable std::mutex mtx;  // mutable — лочим даже в const методах

    void set(const std::string& s) {
        std::lock_guard<std::mutex> lock(mtx);
        data = s;
    }

    std::string get() const {          // const — не меняем данные
        std::lock_guard<std::mutex> lock(mtx);  // без mutable — ошибка компиляции!
        return data;
    }
};

int main() {
    ThreadSafeString s;
    s.set("hello");
    std::cout << s.get() << "\n";
}
```

### Второй типичный пример — кэш

```cpp
struct ExpensiveCalc {
    int data = 10;
    mutable int cache = -1;  // деталь реализации, не часть логического состояния

    int compute() const {    // логически const — данные не меняем
        if (cache == -1) {
            cache = data * data;  // без mutable — ошибка!
        }
        return cache;
    }
};

int main() {
    ExpensiveCalc c;
    std::cout << c.compute() << "\n";  // 100
    std::cout << c.compute() << "\n";  // 100, из кэша
}
```

### Суть `mutable`

Разделяет два понятия:

| | Описание |
|---|---|
| **Физический const** | ни один байт объекта не меняется |
| **Логический const** | наблюдаемое состояние объекта не меняется |

`mutable` говорит: "это поле — деталь реализации, не часть логического состояния". Мьютекс и кэш — типичные примеры.
