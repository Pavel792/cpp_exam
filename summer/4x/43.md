# 43. Дизайн многопоточных приложений

## Deadlock

Дедлок — два потока ждут друг друга вечно:

```cpp
#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx1, mtx2;

void thread1() {
    std::lock_guard<std::mutex> lock1(mtx1);  // захватили mtx1
    std::this_thread::sleep_for(std::chrono::milliseconds(10));
    std::lock_guard<std::mutex> lock2(mtx2);  // ждём mtx2 — его держит thread2!
}

void thread2() {
    std::lock_guard<std::mutex> lock2(mtx2);  // захватили mtx2
    std::this_thread::sleep_for(std::chrono::milliseconds(10));
    std::lock_guard<std::mutex> lock1(mtx1);  // ждём mtx1 — его держит thread1!
}

int main() {
    std::thread t1(thread1);
    std::thread t2(thread2);
    t1.join();  // висим вечно
    t2.join();
}
```

```
thread1: держит mtx1, ждёт mtx2
thread2: держит mtx2, ждёт mtx1  ← круговое ожидание
```

## Recursive mutex

Обычный мьютекс нельзя лочить дважды из одного потока — дедлок.
`std::recursive_mutex` считает вложенность — разлочивается только когда счётчик = 0:

```cpp
#include <mutex>
#include <iostream>

std::recursive_mutex mtx;

void bar() {
    std::lock_guard<std::recursive_mutex> lock(mtx);  // счётчик = 2
    std::cout << "bar\n";
}  // счётчик = 1

void foo() {
    std::lock_guard<std::recursive_mutex> lock(mtx);  // счётчик = 1
    bar();  // ok — тот же поток, счётчик = 2
}  // счётчик = 0 — мьютекс реально свободен

int main() {
    std::thread t(foo);
    t.join();
}
```

`recursive_mutex` — признак плохого дизайна. Лучше разделить API.

## Приватный API без блокировок / публичный с блокировками

Проблема — публичные методы вызывают друг друга и лочат мьютекс дважды:

```cpp
struct Bad {
    std::vector<int> data;
    std::mutex mtx;

    void push(int x) {
        std::lock_guard lock(mtx);
        data.push_back(x);
    }

    void push_twice(int x) {
        push(x);  // лочит mtx
        push(x);  // лочит mtx снова — дедлок!
    }
};
```

**Решение — два слоя:**
- **Приватные `_impl`** — работают с данными, не атомарны сами по себе, вызываются только под уже захваченным мьютексом
- **Публичные** — захватывают мьютекс один раз, вызывают `_impl`

```cpp
#include <vector>
#include <mutex>

struct ThreadSafeVector {
    std::vector<int> data;
    mutable std::mutex mtx;

    // приватные — без блокировок
    void push_impl(int x) { data.push_back(x); }
    int size_impl() const { return data.size(); }

    // публичные — лочат и вызывают приватные
    void push(int x) {
        std::lock_guard lock(mtx);
        push_impl(x);
    }

    int size() const {
        std::lock_guard lock(mtx);
        return size_impl();
    }

    void push_twice(int x) {
        std::lock_guard lock(mtx);
        push_impl(x);  // нет дедлока — мьютекс уже захвачен
        push_impl(x);
    }
};

int main() {
    ThreadSafeVector v;
    v.push(1);
    v.push_twice(2);
}
```

## Избегание взаимных блокировок

### Контроль порядка взятия блокировок

Если все потоки берут мьютексы в **одном порядке** — дедлок невозможен:

```cpp
#include <mutex>
#include <thread>

std::mutex mtx1, mtx2;

void thread1() {
    std::lock_guard lock1(mtx1);  // всегда сначала mtx1
    std::lock_guard lock2(mtx2);  // потом mtx2
}

void thread2() {
    std::lock_guard lock1(mtx1);  // тот же порядок — дедлока нет
    std::lock_guard lock2(mtx2);
}

int main() {
    std::thread t1(thread1);
    std::thread t2(thread2);
    t1.join(); t2.join();
}
```

### `std::scoped_lock` — захватывает несколько мьютексов атомарно

```cpp
#include <mutex>
#include <thread>

std::mutex mtx1, mtx2;

void thread1() {
    std::scoped_lock lock(mtx1, mtx2);  // порядок не важен — безопасно
}

void thread2() {
    std::scoped_lock lock(mtx2, mtx1);  // порядок другой — всё равно безопасно
}

int main() {
    std::thread t1(thread1);
    std::thread t2(thread2);
    t1.join(); t2.join();
}
```

### `unique_lock` + `defer_lock` + `std::lock`

Когда нужна гибкость `unique_lock` (unlock посередине и т.д.):

```cpp
std::mutex mtx1, mtx2;

void foo() {
    std::unique_lock lock1(mtx1, std::defer_lock);  // создали, не лочили
    std::unique_lock lock2(mtx2, std::defer_lock);  // создали, не лочили
    std::lock(lock1, lock2);  // лочим оба атомарно
    // если не смог захватить второй — отпускает первый и пробует снова
}
```

## `thread_local`

У каждого потока **своя копия** переменной — мьютекс не нужен:

```cpp
#include <iostream>
#include <thread>

int global_counter = 0;          // одна на всех — гонка!
thread_local int local_counter = 0;  // у каждого потока своя — безопасно

void foo() {
    for (int i = 0; i < 3; i++) {
        global_counter++;   // гонка
        local_counter++;    // безопасно
    }
    std::cout << "local: " << local_counter << "\n";  // всегда 3
}

int main() {
    std::thread t1(foo);
    std::thread t2(foo);
    t1.join(); t2.join();
    std::cout << "global: " << global_counter << "\n";  // не 6!
}
```

Типичные применения — `errno`-подобные флаги ошибок, кэш per-поток:

```cpp
thread_local int last_error = 0;       // свой errno у каждого потока
thread_local std::vector<int> cache;   // свой кэш — мьютекс не нужен
```

| | Глобальная | `thread_local` |
|---|---|---|
| Копий | одна | по одной на поток |
| Гонки | да | нет |
| Мьютекс нужен | да | нет |
| Потоки видят друг друга | да | нет |

## Частичная потокобезопасность `shared_ptr`

`shared_ptr` хранит два указателя: на объект и на control block (счётчик ссылок):

```
shared_ptr
├── ptr  → [  объект  ]
└── ctrl → [ count=2  ]
```

```cpp
#include <memory>
#include <thread>

int main() {
    auto ptr = std::make_shared<int>(42);

    // ✅ безопасно — счётчик атомарный
    std::thread t1([ptr]{ auto p = ptr; });
    std::thread t2([ptr]{ auto p = ptr; });
    t1.join(); t2.join();

    // ❌ гонка — оба пишут в объект
    std::thread t3([ptr]{ (*ptr)++; });
    std::thread t4([ptr]{ (*ptr)++; });
    t3.join(); t4.join();

    // ❌ гонка — оба меняют один shared_ptr
    std::thread t5([&ptr]{ ptr = std::make_shared<int>(1); });
    std::thread t6([&ptr]{ ptr = std::make_shared<int>(2); });
    t5.join(); t6.join();
}
```

| Операция | Потокобезопасно |
|---|---|
| Копировать разные `shared_ptr` на один объект | ✅ |
| Читать объект из разных потоков | ✅ |
| Писать в объект из разных потоков | ❌ нужен мьютекс |
| Менять один `shared_ptr` из разных потоков | ❌ нужен мьютекс |

## TOCTOU (Time Of Check To Time Of Use)

Между **проверкой** и **использованием** другой поток мог изменить состояние:

```cpp
// поток 1                       // поток 2
if (!queue.empty())              //
                                 // queue.pop() — очередь пуста!
    int x = queue.front();       // UB — очередь пуста
```

**Решение — проверка и использование под одним мьютексом:**

```cpp
#include <queue>
#include <mutex>
#include <iostream>

std::queue<int> q;
std::mutex mtx;

void consumer() {
    std::lock_guard lock(mtx);
    if (!q.empty()) {       // проверили
        int x = q.front();  // использовали — никто не вклинится
        q.pop();
        std::cout << x << "\n";
    }
}

void producer() {
    std::lock_guard lock(mtx);
    q.push(42);
}

int main() {
    std::thread t1(producer);
    std::thread t2(consumer);
    t1.join(); t2.join();
}
```
