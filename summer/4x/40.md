# 40. Базовая многопоточность

## Создание потоков в C++11

```cpp
#include <iostream>
#include <thread>

void foo(int x) {
    std::cout << "поток получил: " << x << "\n";
}

int main() {
    // передача по значению — копируется
    int val = 42;
    std::thread t1(foo, val);

    // передача по ссылке — нужен std::ref, иначе скопируется
    int ref_val = 10;
    std::thread t2(foo, std::ref(ref_val));

    // лямбда
    std::thread t3([val]{ std::cout << "лямбда: " << val << "\n"; });

    t1.join();
    t2.join();
    t3.join();
}
```

**Важно:** если передаёшь по ссылке — объект должен пережить поток. Иначе dangling reference.

## Гонки (race conditions)

### Гонка по данным

`++` **не атомарен** — это три операции на уровне процессора:

```
1. READ  — прочитать значение из памяти в регистр
2. ADD   — прибавить 1
3. WRITE — записать обратно в память
```

Между ними планировщик может переключить поток:

```
Поток 1: READ  counter = 5
Поток 2: READ  counter = 5   ← оба прочитали одно значение!
Поток 1: ADD   reg = 6
Поток 2: ADD   reg = 6
Поток 1: WRITE counter = 6
Поток 2: WRITE counter = 6   ← оба записали 6, должно быть 7 — итерация потеряна
```

```cpp
#include <iostream>
#include <thread>

int counter = 0;

void increment() {
    for (int i = 0; i < 100000; i++)
        counter++;  // гонка! результат непредсказуем
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();
    std::cout << counter << "\n";  // не 200000!
}
```

### Одновременное чтение без записи — безопасно

```cpp
const int data = 42;
// несколько потоков только читают — гонки нет
std::thread t1([&]{ std::cout << data; });
std::thread t2([&]{ std::cout << data; });
t1.join(); t2.join();
```

### Гонка при выводе на экран

`cout` частично потокобезопасен: отдельные вызовы `<<` не перемешаются, но между вызовами может вклиниться другой поток:

```cpp
// может вывести строки в любом порядке и вперемешку
std::thread t1([]{ std::cout << "hello" << " world\n"; });
std::thread t2([]{ std::cout << "foo" << " bar\n"; });
t1.join(); t2.join();
```

## Борьба с гонками

### Мьютекс

Только один поток может находиться в критической секции одновременно:

```cpp
#include <iostream>
#include <thread>
#include <mutex>

int counter = 0;
std::mutex mtx;

void increment() {
    for (int i = 0; i < 100000; i++) {
        mtx.lock();
        counter++;  // критическая секция
        mtx.unlock();
    }
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();
    std::cout << counter << "\n";  // 200000
}
```

Проблема голого `lock/unlock` — если бросит исключение, мьютекс не освободится → дедлок.

### `lock_guard` — простая RAII обёртка

```cpp
void increment() {
    for (int i = 0; i < 100000; i++) {
        std::lock_guard<std::mutex> lock(mtx);  // лочит при создании
        counter++;
    }  // анлочит при выходе из {} — даже при исключении
}
```

### `unique_lock` — гибкая RAII обёртка

```cpp
#include <mutex>

std::mutex mtx;

void foo() {
    std::unique_lock<std::mutex> lock(mtx);  // сразу лочит
    // ... критическая секция
    lock.unlock();   // можно разлочить раньше конца блока
    // ... не критическая секция
    lock.lock();     // можно залочить снова
}  // разлочит при выходе если залочен

// отложенный лок
void bar() {
    std::unique_lock<std::mutex> lock(mtx, std::defer_lock);  // не лочит сразу
    // ... что-то делаем
    lock.lock();  // лочим когда нужно
}
```

### Атомарные операции

Для простых типов мьютекс избыточен — достаточно `std::atomic`:

```cpp
#include <atomic>
#include <thread>
#include <iostream>

std::atomic<int> counter = 0;

void increment() {
    for (int i = 0; i < 100000; i++)
        counter++;  // READ+ADD+WRITE неделимо
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();
    std::cout << counter << "\n";  // 200000
}
```

Мьютекс нужен когда операция сложная или затрагивает **несколько переменных сразу**:

```cpp
{
    std::lock_guard lock(mtx);
    x++;
    y++;  // x и y обновляются вместе атомарно
}
```

## Joinable / Detached потоки

### `join()` — главный поток ждёт

```cpp
#include <iostream>
#include <thread>

int main() {
    std::thread t([]{ std::cout << "поток работает\n"; });
    t.join();  // main ждёт пока t завершится
    std::cout << "поток завершён\n";
}
```

**Важно:** если `std::thread` уничтожается пока joinable (без `join()` или `detach()`) — `std::terminate`:

```cpp
int main() {
    std::thread t([]{ /* ... */ });
}  // забыли join/detach — terminate!
```

### `detach()` — поток живёт самостоятельно

```cpp
int main() {
    std::thread t([]{ std::cout << "работаю сам\n"; });
    t.detach();  // отпустили — main не ждёт
    // если main завершится раньше — поток убьётся вместе с программой
}
```

**Опасность detach** — dangling reference если поток обращается к локальным переменным:

```cpp
int main() {
    int x = 42;
    std::thread t([&x]{ std::cout << x << "\n"; });  // dangling если main вышел!
    t.detach();
}  // x уничтожен, поток ещё работает — UB
```

### Проверка состояния

```cpp
std::thread t([]{ /* ... */ });
t.joinable();  // true
t.join();
t.joinable();  // false
```

### RAII для потоков

`std::thread` не делает `join()` в деструкторе — делаем сами:

```cpp
#include <thread>

struct JoinThread {
    std::thread t;
    explicit JoinThread(std::thread t) : t(std::move(t)) {}
    ~JoinThread() {
        if (t.joinable()) t.join();
    }
};

int main() {
    JoinThread t(std::thread([]{ /* ... */ }));
}  // деструктор сам вызовет join()
```

В C++20 это уже в стандарте — `std::jthread` делает `join()` в деструкторе автоматически.
