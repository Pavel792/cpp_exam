# 41. TCP-соединения при помощи блокирующего ввод-вывода в Boost::Asio

## Что характеризует TCP-соединение

TCP-соединение — это **две пары** (хост, порт):

```
Клиент: 192.168.1.5 : 54321  ←→  Сервер: 93.184.216.34 : 12345
         случайный порт                    фиксированный порт
```

**Почему порт сервера фиксирован?** Клиент должен знать куда подключаться заранее. HTTP — 80, HTTPS — 443, SSH — 22. Если порт случайный — клиент просто не найдёт сервер.

**Почему порт клиента случаен?** Серверу всё равно откуда пришёл клиент — он отвечает на тот адрес откуда пришёл запрос. ОС выдаёт свободный порт автоматически.

## Эхо-сервер на `boost::asio::ip::tcp::iostream`

### Сервер

```cpp
#include <boost/asio.hpp>
#include <iostream>
#include <string>

int main() {
    boost::asio::io_context ctx;

    // acceptor слушает входящие соединения на порту 12345
    boost::asio::ip::tcp::acceptor acceptor(
        ctx,
        boost::asio::ip::tcp::endpoint(boost::asio::ip::tcp::v4(), 12345)
    );

    while (true) {
        boost::asio::ip::tcp::iostream stream;
        acceptor.accept(*stream.rdbuf());  // блокируется пока не придёт клиент

        std::cout << "клиент: " << stream.rdbuf()->remote_endpoint() << "\n";

        std::string line;
        while (std::getline(stream, line)) {
            stream << line << "\n";  // эхо — отправляем строку обратно
        }
        // клиент отключился — ждём следующего
    }
}
```

### Клиент

Клиент просто подключается к уже существующему серверу — не знает ничего про его внутренности. `tcp::iostream` — обычный C++ stream поверх TCP, работает как `std::fstream`, только вместо файла — сетевое соединение:

```cpp
#include <boost/asio.hpp>
#include <iostream>
#include <string>

int main() {
    // 1. резолвит "localhost" → 127.0.0.1
    // 2. ОС выдаёт клиенту случайный порт (например 54321)
    // 3. устанавливает соединение — блокируется пока сервер не ответит
    boost::asio::ip::tcp::iostream stream("localhost", "12345");

    if (!stream) {  // сервер мог быть недоступен
        std::cout << "не удалось подключиться\n";
        return 1;
    }

    // \n важен — сервер читает через getline и ждёт \n как конец сообщения
    // без \n сервер будет ждать вечно
    stream << "hello\n";

    // блокируемся пока сервер не пришлёт ответ
    std::string response;
    std::getline(stream, response);
    std::cout << "сервер ответил: " << response << "\n";
}
```

### Полный цикл одного обмена

```
Клиент                          Сервер
  |                               |
  |---[ SYN ]-------------------->|  установка соединения
  |<--[ SYN-ACK ]-----------------|
  |---[ ACK ]-------------------->|
  |                               |
  | stream << "hello\n"           |
  |---[ hello\n ]---------------->|  getline читает "hello"
  |                               |  stream << "hello\n"
  |<--[ hello\n ]-----------------|
  | getline читает "hello"        |
```

## `local_endpoint()` vs `remote_endpoint()`

```
Клиент 192.168.1.5:54321  ←→  Сервер 93.184.216.34:12345
```

| | На клиенте | На сервере |
|---|---|---|
| `local_endpoint()` | 192.168.1.5:54321 | 93.184.216.34:12345 |
| `remote_endpoint()` | 93.184.216.34:12345 | 192.168.1.5:54321 |

`local` — это **ты сам**. `remote` — это **собеседник**.

## TCP передаёт байты, не сообщения

Ключевое свойство TCP — поток байтов без границ сообщений. За один вызов чтения можно получить:

```
Отправили:  [  сообщение 1  ][  сообщение 2  ]

Получили за один read():
  вариант А: [  сообщение 1  ]                    — только первое
  вариант Б: [  сообщение 1  ][ сообще ]          — первое и кусок второго
  вариант В: [  сообщение 1  ][  сообщение 2  ]   — оба сразу
```

**Решение — разделять сообщения самостоятельно.** Два популярных способа:

```cpp
// 1. Разделитель \n — читаем по строкам
std::string line;
while (std::getline(stream, line)) {
    process(line);  // line — одно полное сообщение
}

// 2. Длина в заголовке — сначала читаем размер, потом тело
uint32_t len;
stream.read(reinterpret_cast<char*>(&len), sizeof(len));  // 4 байта длины
std::string msg(len, '\0');
stream.read(msg.data(), len);  // ровно len байт тела
```

`std::getline` удобен для простых протоколов — `\n` как граница сообщения решает проблему автоматически.

## Почему блокирующий

`tcp::iostream` — намеренно простой блокирующий интерфейс. Пока данные не пришли — поток стоит и ждёт. Просто, но не масштабируется — **один поток обслуживает одно соединение**. `boost::asio` умеет и асинхронный ввод-вывод через колбэки/корутины, но это уже другая история.
