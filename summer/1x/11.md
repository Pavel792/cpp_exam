# 11. Реализация своего std::vector

## Placement new, выравнивание, явный вызов деструктора

Обычный `new` делает два дела: выделяет память и конструирует объект.
В векторе эти шаги нужно разделить — память выделяем заранее, объекты создаём потом.

```cpp
// Обычный new — нельзя использовать в векторе
T* p = new T();  // выделяет память И конструирует

// Placement new — конструирует объект в уже выделенной памяти
T* data = Alloc().allocate(C);  // только выделяем память
new (data + i) T(value);        // конструируем объект по адресу data+i
```

Раз конструировали вручную — деструктор тоже вызываем вручную:

```cpp
data[i].~T();  // явный вызов деструктора
```

Нельзя написать `delete data` — это вызвало бы деструктор для всех `C` элементов,
а сконструированы только первые `len`.

### Выравнивание: `alignof` / `alignas`

Процессор читает память блоками. Если объект лежит не на «правильном» адресе — это медленно или UB.

```cpp
alignof(char)    // 1 — можно класть на любой адрес
alignof(int)     // 4 — нужен адрес кратный 4
alignof(double)  // 8 — нужен адрес кратный 8

alignas(16) char buf[64];  // buf будет на адресе кратном 16
```

Если выделять память через `char[]` без `alignas` — placement new будет UB:

```cpp
// ПЛОХО
char buf[sizeof(T) * 10];
new (buf) T();  // UB если alignof(T) > 1

// ХОРОШО
alignas(T) char buf[sizeof(T) * 10];
new (buf) T();  // OK
```

В `minivector` проблемы нет — `Alloc().allocate(C)` уже возвращает память
выровненную под `T`.

## Время жизни объектов (lifetime)

Объект существует с момента конструирования до момента вызова деструктора.
До и после — обращаться к нему нельзя (UB).

```cpp
T* data = Alloc().allocate(C);  // память есть, объектов НЕТ
data[0].~T();                   // UB! объект не существует

new (data) T();   // теперь data[0] существует
data[0].~T();     // OK — заканчиваем lifetime

data[0].foo();    // UB! lifetime закончен
```

В `minivector` это контролируется через `len` — только первые `len` элементов
реально сконструированы. Обращение за `len` — UB.

## Ref-qualifiers: `&`, `&&`, перегрузка по ним и по const

Как `const` позволяет перегружать методы по константности объекта,
так `&` и `&&` позволяют перегружать по категории значения объекта (`*this`).

```cpp
void foo() &;        // вызывается только на lvalue
void foo() &&;       // вызывается только на rvalue
void foo() const &;  // вызывается на const lvalue
```

Пример из `minivector`:

```cpp
void push_back(const T &value) &noexcept { ... }  // только на lvalue-векторе
void pop_back() &noexcept { ... }                  // только на lvalue-векторе
```

Запрет вызова на временном объекте — разумен: добавлять элементы во временный
вектор, который тут же уничтожится, бессмысленно.

## Перегрузка `operator[]`

Три перегрузки — по двум осям: `const` и ref-qualifier:

```cpp
T &operator[](std::size_t i) &noexcept {
    return data[i];
}

const T &operator[](std::size_t i) const &noexcept {
    return data[i];
}

T &&operator[](std::size_t i) &&noexcept {
    return std::move(data[i]);
}
```

Какая вызывается:

```cpp
minivector<std::string> v;
const minivector<std::string> cv = v;

v[0];             // v — non-const lvalue → первая (&)
cv[0];            // cv — const lvalue    → вторая (const &)
std::move(v)[0];  // xvalue              → третья (&&)
```

Третья перегрузка нужна чтобы при move вектора элементы тоже перемещались, а не копировались.

### `const_cast` для уменьшения дублирования

Если логика сложная — дублировать её в двух перегрузках плохо.
Решение: реальный код в `const`-версии, non-const вызывает её и снимает `const`:

```cpp
const T& operator[](std::size_t i) const & {
    assert(i < len);
    return data[i];  // реальная логика здесь
}

T& operator[](std::size_t i) & {
    return const_cast<T&>(
        static_cast<const minivector&>(*this)[i]
    );
}
```

Два каста:
- `static_cast<const minivector&>(*this)` — добавляем `const` чтобы попасть в нужную перегрузку
- `const_cast<T&>(...)` — снимаем `const` с результата

Безопасно, потому что non-const версия вызывается только на non-const объекте.

**Нельзя наоборот** — вызывать non-const из const и снимать `const` через `const_cast`.
Если объект реально константный — это UB.

## Перегрузка `push_back` для `const T&` и `T&&`

Три способа принять аргумент:

```cpp
// 1. По const T& — всегда копирует
void push_back(const T& value) { new (...) T(value); }
// push_back(std::move(s)) → copy, хотя мог бы быть move ❌

// 2. По значению — лишний move для rvalue
void push_back(T value) { new (...) T(std::move(value)); }
// push_back(s)            → 1 copy + 1 move
// push_back(std::move(s)) → 2 move

// 3. Две перегрузки — оптимально ✅
void push_back(const T& value) { new (...) T(value); }        // 1 copy
void push_back(T&& value)      { new (...) T(std::move(value)); } // 1 move
```

| Способ | lvalue | rvalue |
|---|---|---|
| `const T&` | 1 copy | 1 copy ❌ |
| `T` по значению | 1 copy + 1 move | 2 move |
| две перегрузки | 1 copy ✅ | 1 move ✅ |

Альтернатива двум перегрузкам — perfect forwarding через шаблон (отдельная тема).

## Трудности с базовой гарантией исключений

Проблема: конструируем объекты один за другим, в середине может вылететь исключение.
Уже сконструированные объекты надо уничтожить, иначе утечка ресурсов.

```cpp
minivector(std::size_t n) : data(Alloc().allocate(C)), len(0) {
    for (; len < n; ++len) {
        try {
            new (data + len) T();  // может бросить исключение
        } catch (...) {
            while (len > 0) {
                pop_back();        // уничтожаем уже сконструированные
            }
            Alloc().deallocate(data, C);  // освобождаем память
            throw;                 // пробрасываем исключение дальше
        }
    }
}
```

`len` в момент исключения равен числу уже сконструированных объектов.
`pop_back` вызывает деструктор и уменьшает `len`. Так обеспечивается базовая гарантия:
при исключении объект не создаётся, память не утекает.

## Полная реализация minivector

```cpp
#ifndef MINIVECTOR_H_
#define MINIVECTOR_H_

#include <cassert>
#include <memory>
#include <utility>

namespace ex_minivector {

template <typename T, typename Alloc = std::allocator<T>, std::size_t C = 10>
struct minivector {
private:
    T *data;
    std::size_t len;

public:
    minivector() : data(Alloc().allocate(C)), len(0) {
    }

    minivector &operator=(const minivector &) = delete;
    minivector &operator=(minivector &&) = delete;

    ~minivector() {
        for (std::size_t i = 0; i < len; i++) {
            data[i].~T();
        }
        Alloc().deallocate(data, C);
    }

    std::size_t size() const noexcept {
        return len;
    }

    T &operator[](std::size_t i) &noexcept {
        return data[i];
    }

    const T &operator[](std::size_t i) const &noexcept {
        return data[i];
    }

    T &&operator[](std::size_t i) &&noexcept {
        return std::move(data[i]);
    }

    minivector(std::size_t n) : data(Alloc().allocate(C)), len(0) {
        for (; len < n; ++len) {
            try {
                new (data + len) T();
            } catch (...) {
                while (len > 0) {
                    pop_back();
                }
                Alloc().deallocate(data, C);
                throw;
            }
        }
    }

    void pop_back() &noexcept {
        assert(len > 0);
        data[--len].~T();
    }

    void push_back(const T &value) &noexcept {
        assert(len < C);
        new (&data[len++]) T(value);
    }

    void push_back(T &&value) &noexcept {
        assert(len < C);
        new (&data[len++]) T(std::move(value));
    }

    minivector(std::size_t n, const T &v) : data(Alloc().allocate(C)), len(n) {
        for (std::size_t i = 0; i < n; i++) {
            new (data + i) T(v);
        }
    }

    minivector(const minivector &other)
        : data(Alloc().allocate(C)), len(other.len) {
        for (std::size_t i = 0; i < len; i++) {
            new (data + i) T(other[i]);
        }
    }

    minivector(minivector &&other)
        : data(std::exchange(other.data, Alloc().allocate(C))),
          len(std::exchange(other.len, 0)) {
    }
};

}  // namespace ex_minivector

#endif  // MINIVECTOR_H_
```
