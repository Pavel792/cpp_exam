# 10. Move-семантика и copy elision

## Категории значений

Каждое выражение в C++ имеет **тип** и **категорию значения**. Три базовых категории (листья):

```
        expression
        /         \
    glvalue       rvalue
    /     \       /    \
lvalue   xvalue  prvalue
```

### lvalue
Объект с именем и адресом, из которого нельзя украсть по умолчанию.
```cpp
int x = 5;
x        // lvalue
arr[0]   // lvalue
++x      // lvalue — префиксный инкремент возвращает сам объект
```

### prvalue
Временный объект без адреса, из которого можно украсть (он всё равно умрёт).
```cpp
42                    // prvalue
x + 1                 // prvalue
std::string("hello")  // prvalue
x++                   // prvalue — постфиксный инкремент возвращает копию
```

### xvalue
Объект с адресом, из которого явно разрешили украсть. Получается через `std::move`.
```cpp
std::move(x)  // xvalue
```

### glvalue и rvalue — группы, не листья
Конкретное выражение никогда не бывает «просто glvalue» или «просто rvalue» —
оно всегда lvalue, xvalue или prvalue. glvalue и rvalue нужны чтобы говорить о группах сразу.

| | Есть адрес | Можно украсть |
|---|---|---|
| lvalue | ✅ | ❌ |
| xvalue | ✅ | ✅ |
| prvalue | ❌ | ✅ |
| **glvalue** = lvalue + xvalue | ✅ | — |
| **rvalue** = xvalue + prvalue | — | ✅ |

## lvalue-ссылки и rvalue-ссылки: что к чему привязывается

```cpp
T&   // lvalue-ссылка  → привязывается к lvalue
T&&  // rvalue-ссылка  → привязывается к rvalue (xvalue или prvalue)
```

```cpp
int x = 5;

int& a = x;             // ✅ lvalue к lvalue
int& b = 42;            // ❌ lvalue-ссылка не берёт rvalue

int&& c = 42;           // ✅ rvalue к prvalue
int&& d = std::move(x); // ✅ rvalue к xvalue
int&& e = x;            // ❌ rvalue-ссылка не берёт lvalue

const int& f = 42;      // ✅ исключение: const lvalue-ссылка берёт rvalue
```

| | `T&` | `const T&` | `T&&` |
|---|---|---|---|
| lvalue | ✅ | ✅ | ❌ |
| xvalue | ❌ | ✅ | ✅ |
| prvalue | ❌ | ✅ | ✅ |

Категория результата вызова функции зависит от **типа возврата**:

| Тип возврата | Категория вызова |
|---|---|
| `T` | prvalue |
| `T&` | lvalue |
| `T&&` | xvalue |

## Move constructor и move assignment: сигнатура

```cpp
MyClass(const MyClass& other);      // copy constructor
MyClass(MyClass&& other) noexcept;  // move constructor — не const!

MyClass& operator=(const MyClass& other);     // copy assignment
MyClass& operator=(MyClass&& other) noexcept; // move assignment — не const!
```

### Почему не `const T&&`

Move constructor должен **опустошить** источник — забрать ресурсы и обнулить:
```cpp
MyClass(MyClass&& other) noexcept {
    data = other.data;
    other.data = nullptr;  // модифицируем other — через const нельзя!
    other.size = 0;
}
```

### Разрешение перегрузок

Компилятор выбирает наиболее точное совпадение. `T&&` точнее совпадает с rvalue чем `const T&`:

```cpp
MyClass a;
MyClass b = a;             // a — lvalue → copy constructor
MyClass c = std::move(a);  // xvalue → move constructor
MyClass d = MyClass();     // prvalue → move constructor (или copy elision)
```

| Аргумент | `const T&` | `T&&` | Победитель |
|---|---|---|---|
| lvalue | ✅ | ❌ | copy |
| xvalue | ✅ | ✅ | move (точнее) |
| prvalue | ✅ | ✅ | move (точнее) |

## std::move

`std::move` **не перемещает данные**. Это просто каст — меняет категорию выражения с lvalue на xvalue. Реальное перемещение происходит потом в move constructor/assignment.

```cpp
std::string s = "hello";
std::move(s);              // s всё ещё "hello", ничего не произошло
std::string t = std::move(s); // вот здесь данные реально переехали
```

### Реализация

```cpp
template<typename T>
std::remove_reference_t<T>&& my_move(T&& t) noexcept {
    return static_cast<std::remove_reference_t<T>&&>(t);
}
```

`remove_reference_t` убирает ссылку из типа, `&&` делает rvalue-ссылку. Итог — xvalue.

### Где используется

```cpp
// 1. Передать владение в контейнер
std::vector<std::string> v;
std::string s = "hello";
v.push_back(std::move(s));

// 2. Передать владение в функцию
void process(std::string&& s);
process(std::move(s));

// 3. Внутри move constructor — other внутри тела является lvalue!
MyClass(MyClass&& other) {
    data = std::move(other.data);  // без std::move был бы copy
}
```

## Copy elision, RVO, NRVO

Компилятор создаёт объект сразу в нужном месте памяти, не вызывая copy/move вообще.

### RVO (Return Value Optimization)
Функция возвращает prvalue — объект создаётся сразу у вызывающего. Гарантировано с C++17.

```cpp
MyClass foo() {
    return MyClass();  // RVO — никакого move/copy
}
MyClass a = foo();
```

### NRVO (Named Return Value Optimization)
Функция возвращает именованную переменную. Не гарантировано, но компиляторы делают при оптимизации.

```cpp
MyClass bar() {
    MyClass result;
    result.push_back(1);
    return result;  // NRVO
}
MyClass a = bar();
```

Если NRVO не сработало — компилятор автоматически сделает move, не copy.

| | RVO | NRVO |
|---|---|---|
| Что возвращается | prvalue (безымянный) | именованная переменная |
| Гарантировано | ✅ C++17 | ❌ |
| Если не сработало | — | автоматически move |

## Когда (не) надо писать `return std::move(foo)`

### ❌ Никогда не пиши при возврате локальной переменной

```cpp
MyClass foo() {
    MyClass result;
    return std::move(result);  // ПЛОХО — ломает NRVO
}
```

`std::move(result)` превращает result в xvalue — NRVO к xvalue неприменимо.
Вместо элизии получаем move constructor — хуже.

Просто пиши `return result` — компилятор сам попробует NRVO, потом автоматически move.

### ✅ Уместно для параметров и членов класса

```cpp
// параметр — NRVO неприменимо
MyClass process(MyClass input) {
    return std::move(input);  // ✅
}

// член класса
MyClass consume() {
    return std::move(field);  // ✅
}
```

### Правило

> Возвращаешь **локальную переменную** — никогда не пиши `std::move`.  
> Возвращаешь параметр или член класса — можно.
