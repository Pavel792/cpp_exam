# 30. Исключения — основы

## Инвариант объекта, предусловия и постусловия

**Инвариант объекта** — условие, которое всегда истинно для корректно созданного объекта.  
Например, у `std::vector`: `size <= capacity`.

**Предусловие** — что должно быть истинно *до* вызова функции/конструктора.  
**Постусловие** — что гарантируется *после*.

- **Конструктор** — устанавливает инвариант. Если не смог — бросает исключение.
- **Деструктор** — освобождает ресурсы. Вызывается *всегда*, даже при раскрутке стека.

```cpp
struct File {
    FILE* f;

    File(const char* name) {
        f = fopen(name, "r");
        if (!f) throw std::runtime_error("cannot open file"); // предусловие нарушено
        // инвариант установлен: f != nullptr
    }

    ~File() {
        fclose(f); // вызовется всегда, даже при исключении снаружи
    }
};
```

## Ошибки программирования vs ошибки окружения

| | Ошибка программирования | Ошибка окружения |
|---|---|---|
| Причина | Баг в коде | Внешние условия |
| Предсказуемость | Нет | Да |
| Стратегия | `assert` / `abort` | исключения / коды возврата |
| Пример | разыменование `nullptr` | файл не найден |

**Ошибки программирования** — нельзя предсказать и обработать в рантайме. Стратегия: аварийное завершение.

```cpp
void foo(int* ptr) {
    assert(ptr != nullptr);  // ошибка программиста — сразу падаем
    *ptr = 42;
}
```

**Ошибки окружения** — внешние условия, которые можно предсказать и обработать.

```cpp
std::ifstream f("data.txt");
if (!f) throw std::runtime_error("file not found");
```

## Техники обработки ошибок без исключений

### 1. Код возврата (C-стиль)

Функция возвращает код ошибки, результат пишет в буфер через указатель:

```c
int divide(int a, int b, int* result) {
    if (b == 0) return -1;  // код ошибки
    *result = a / b;
    return 0;               // успех
}
```

### 2. Глобальная переменная (`errno`)

```c
FILE* f = fopen("file.txt", "r");
if (!f) {
    printf("Error: %d\n", errno);  // ENOENT, EACCES и т.д.
}
```

Минус — хрупко, хотя в современных реализациях `errno` thread-local.

### 3. Состояние объекта

Объект хранит флаг ошибки внутри — как `std::ifstream`:

```cpp
std::ifstream f("file.txt");
if (!f) { /* ошибка */ }
```

### 4. `assert` и `exit`

```cpp
assert(ptr != nullptr);  // падает в debug-сборке
exit(1);                 // безусловное завершение
```

**Почему исключения лучше:** коды возврата легко проигнорировать. Исключение — нельзя.

## Синтаксис `try` / `catch` / `throw`

```cpp
try {
    throw std::runtime_error("something went wrong");
} catch (const std::runtime_error& e) {
    // конкретный тип
    std::cout << e.what();
} catch (const std::exception& e) {
    // любое стандартное исключение
} catch (...) {
    // вообще всё
}
```

### Несколько `catch` подряд

Проверяются **сверху вниз** — более конкретные типы должны быть выше:

```cpp
try {
    throw std::runtime_error("oops");
} catch (const std::runtime_error& e) {  // сначала конкретный
    std::cout << "runtime_error";
} catch (const std::exception& e) {      // потом общий
    std::cout << "exception";
}
```

### Вложенные `try/catch`

```cpp
try {
    try {
        throw std::runtime_error("inner");
    } catch (const std::logic_error& e) {
        // не поймает runtime_error — летит дальше
    }
} catch (const std::runtime_error& e) {
    // поймается здесь
}
```

### Наследование и `catch`

```
std::exception
├── std::runtime_error
│   ├── std::range_error
│   └── std::overflow_error
└── std::logic_error
    ├── std::invalid_argument
    └── std::out_of_range
```

`catch` ловит по иерархии наследования — `catch (const std::logic_error&)` поймает `out_of_range`.  
Внутри `catch` при вызове `e.what()` уже работает полиморфизм (виртуальная функция).

## Слайсинг (Object Slicing)

Срезка — когда объект производного класса копируется в объект базового. Лишняя часть теряется:

```cpp
Derived d;
Base b = d;      // срезка! данные Derived потеряны, виртуальные методы не работают
```

**Проблема при исключениях:**

```cpp
} catch (std::exception e) {    // по значению — срезка!
    e.what();                   // вызовется exception::what(), а не производный
}
```

**Решение — всегда ловить по `const&`:**

```cpp
} catch (const std::exception& e) {  // ссылка — срезки нет, полиморфизм работает
    e.what();
}
```

> Ловить по обычной `&` тоже легально — исключение хранится в специальной области памяти (exception storage), а не на стеке. Но `const&` — хороший тон: сигнализирует, что мы не меняем исключение.

## Раскрутка стека (Stack Unwinding)

**Стек** — область памяти с локальными переменными и фреймами вызовов (LIFO).

При броске исключения, пока оно летит к `catch`:
- каждый фрейм снимается со стека
- для всех локальных объектов вызываются деструкторы
- процесс останавливается когда находится подходящий `catch`

```cpp
void bar() { throw std::runtime_error("oops"); }

void foo() {
    std::string s = "hello";  // локальный объект
    bar();                    // исключение летит вверх
    // s.~string() вызовется автоматически при раскрутке
}

int main() {
    try { foo(); }
    catch (const std::exception& e) { /* поймано здесь */ }
}
```

## RAII (Resource Acquisition Is Initialization)

**Идея:** захватывай ресурс в конструкторе, освобождай в деструкторе.  
Деструктор вызывается при любом выходе из `{}` — при `return`, исключении, нормальном завершении.

```cpp
struct File {
    FILE* f;
    File(const char* name) {
        f = fopen(name, "r");
        if (!f) throw std::runtime_error("cannot open");
    }
    ~File() { fclose(f); }  // вызовется всегда
};

void foo() {
    File f("data.txt");  // ресурс захвачен
    // исключение, return, нормальный выход — ~File() вызовется в любом случае
}
```

Без RAII — легко забыть `fclose` при исключении или досрочном `return`.

### Стандартные RAII обёртки

| Ресурс | Обёртка |
|---|---|
| Память | `std::unique_ptr`, `std::shared_ptr` |
| Файл | `std::ifstream`, `std::ofstream` |
| Мьютекс | `std::lock_guard`, `std::unique_lock` |

```cpp
void foo() {
    auto ptr = std::make_unique<int>(42);  // память освободится автоматически
    std::lock_guard<std::mutex> lock(mtx); // мьютекс отпустится при выходе из {}
}
```

### Деструктор не должен бросать исключения

Если деструктор бросит исключение во время раскрутки стека — два исключения одновременно → `std::terminate`:

```cpp
struct Bad {
    ~Bad() { throw std::runtime_error("oops"); }  // никогда так не делай!
};
```

Деструкторы по умолчанию `noexcept` в C++11. Всегда соблюдай это правило.
