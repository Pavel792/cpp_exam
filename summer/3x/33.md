# 33. Исключения — необычная обработка

## Непойманные исключения и раскрутка стека

Если исключение не поймано нигде — вызывается `std::terminate`.
Раскрутка стека **не гарантирована** — деструкторы могут не вызваться:

```cpp
#include <iostream>
#include <stdexcept>

struct Foo {
    ~Foo() { std::cout << "~Foo" << std::endl; }  // может не вызваться!
};

int main() {
    Foo f;
    throw std::runtime_error("непоймано");  // terminate, ~Foo под вопросом
}
```

Если поймать — деструктор гарантированно вызовется:

```cpp
#include <iostream>
#include <stdexcept>

struct Foo {
    ~Foo() { std::cout << "~Foo" << std::endl; }
};

int main() {
    try {
        Foo f;
        throw std::runtime_error("поймано");
    } catch (...) {
        std::cout << "поймали" << std::endl;
    }
    // выведет: ~Foo, потом поймали
}
```

## `catch (...)` — ловит всё

```cpp
#include <iostream>

int main() {
    try {
        throw 42;  // можно бросить что угодно, даже int
    } catch (...) {
        std::cout << "поймали что-то" << std::endl;
        // не знаем что именно — нельзя обратиться к объекту
    }
}
```

## Перебрасывание — `throw;`

`throw;` без аргумента — пробрасывает текущее исключение **без копирования**.
`throw e` — создаёт новое исключение типа переменной `e` — происходит **слайсинг**:

```cpp
#include <iostream>
#include <stdexcept>

int main() {
    try {
        try {
            throw std::runtime_error("oops");
        } catch (std::exception& e) {
            // throw e;  // плохо — слайсинг! бросается копия типа std::exception
            throw;       // правильно — бросается оригинал runtime_error
        }
    } catch (const std::runtime_error& e) {
        std::cout << "runtime_error: " << e.what() << std::endl;  // поймается с throw;
        // с throw e — не поймалось бы здесь, улетело бы в catch(exception)
    } catch (const std::exception& e) {
        std::cout << "exception: " << e.what() << std::endl;
    }
}
```

## `exception_ptr`

`exception_ptr` — умный указатель на исключение. Позволяет сохранить исключение и перебросить позже — в другом месте или другом потоке.

```cpp
#include <iostream>
#include <stdexcept>
#include <exception>

int main() {
    std::exception_ptr eptr;  // пустое состояние — как nullptr

    // Сохраняем исключение
    try {
        throw std::runtime_error("сохранённое исключение");
    } catch (...) {
        eptr = std::current_exception();  // захватываем текущее
    }

    // Проверяем пустое состояние
    if (eptr) {
        try {
            std::rethrow_exception(eptr);  // перебрасываем позже
        } catch (const std::exception& e) {
            std::cout << "поймали позже: " << e.what() << std::endl;
        }
    }
}
```

### Создание без `catch`

```cpp
#include <iostream>
#include <stdexcept>
#include <exception>

int main() {
    // Создаём exception_ptr без throw/catch
    std::exception_ptr eptr = std::make_exception_ptr(
        std::runtime_error("создали напрямую")
    );

    try {
        std::rethrow_exception(eptr);
    } catch (const std::exception& e) {
        std::cout << e.what() << std::endl;
    }

    // Обнуляем
    eptr = nullptr;
    if (!eptr) std::cout << "пустой" << std::endl;
}
```

### Главный сценарий — передача исключений между потоками

```cpp
#include <iostream>
#include <stdexcept>
#include <exception>
#include <thread>

std::exception_ptr eptr;

void worker() {
    try {
        throw std::runtime_error("ошибка в потоке");
    } catch (...) {
        eptr = std::current_exception();  // сохраняем из рабочего потока
    }
}

int main() {
    std::thread t(worker);
    t.join();

    if (eptr) {
        try {
            std::rethrow_exception(eptr);  // перебрасываем в главном потоке
        } catch (const std::exception& e) {
            std::cout << "поймали в main: " << e.what() << std::endl;
        }
    }
}
```

Без `exception_ptr` исключение из потока поймать невозможно — вызовется `terminate`.

## Function-try-block

Оборачивает **всю функцию целиком**, включая список инициализации конструктора.

### Обычная функция — синтаксический сахар

```cpp
#include <iostream>
#include <stdexcept>

// Обычный способ и function-try-block — одно и то же для функций
int foo(int x) try {
    if (x == 0) throw std::runtime_error("ноль");
    return x;
} catch (const std::exception& e) {
    std::cout << "поймали в foo: " << e.what() << std::endl;
    return -1;  // исключение проглочено — можно вернуть значение
}

int main() {
    std::cout << foo(0) << std::endl;   // поймали в foo: ноль, -1
    std::cout << foo(42) << std::endl;  // 42
    std::cout << "продолжаем" << std::endl;  // выведется — исключение проглочено
}
```

### Конструктор — главный смысл

Обычный `try` внутри конструктора не поймает исключения из списка инициализации.
Function-try-block — поймает:

```cpp
#include <iostream>
#include <stdexcept>

struct Base {
    Base() { throw std::runtime_error("ошибка в Base"); }
};

struct Foo : Base {
    int x;

    // Function-try-block — ловит исключения из списка инициализации
    Foo(int x) try : Base(), x(x) {
        std::cout << "тело конструктора\n";  // не выведется
    } catch (const std::exception& e) {
        std::cout << "поймали в конструкторе: " << e.what() << std::endl;
        // исключение перебросится автоматически после catch!
    }
};

int main() {
    try {
        Foo f(1);
    } catch (const std::exception& e) {
        std::cout << "поймали в main: " << e.what() << std::endl;
    }
}
```

## Невозможность обращения к полям в catch конструктора

В `catch` конструктора нельзя обращаться к полям — объект не создан:

```cpp
struct Foo {
    int x;
    std::string name;

    Foo(int x) try : x(x), name("hello") {
    } catch (...) {
        // x и name — UB! объект не был создан до конца
        // std::cout << x;     // UB
        // std::cout << name;  // UB
    }
};
```

## Возможность отменить выброс исключения

| Где | Можно проглотить? |
|---|---|
| Обычная функция | ✅ да — просто не пишем `throw` |
| Метод | ✅ да |
| Конструктор | ❌ нет — исключение перебрасывается автоматически |
| Деструктор | ❌ нет — исключение перебрасывается автоматически (+ `noexcept` → `terminate`) |

```cpp
#include <iostream>
#include <stdexcept>

// Конструктор — нельзя проглотить
struct Foo {
    Foo() try {
        throw std::runtime_error("oops");
    } catch (...) {
        // компилятор автоматически добавляет throw; в конце
        // объект не создан — возвращать его бессмысленно
    }
};

int main() {
    try {
        Foo f;
    } catch (...) {
        std::cout << "всё равно поймали" << std::endl;  // выведется!
    }
}
```
