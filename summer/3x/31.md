# 31. Исключения — детали

## Базовые классы исключений

```
std::exception
├── std::runtime_error      — ошибки окружения (в рантайме)
│   ├── std::range_error
│   ├── std::overflow_error
│   └── std::underflow_error
├── std::logic_error        — ошибки логики программы
│   ├── std::invalid_argument
│   ├── std::out_of_range
│   └── std::length_error
├── std::bad_alloc          — не удалось выделить память (new)
└── std::bad_cast           — неудачный dynamic_cast по ссылке
```

```cpp
#include <iostream>
#include <stdexcept>
#include <typeinfo>

struct Base { virtual ~Base() = default; };
struct Derived : Base {};

int main() {
    // std::exception — базовый класс, what() возвращает сообщение
    try {
        throw std::runtime_error("что-то пошло не так");
    } catch (const std::exception& e) {
        std::cout << "exception: " << e.what() << "\n";
    }

    // std::bad_alloc — не удалось выделить память
    try {
        int* p = new int[999999999999LL];
        (void)p;
    } catch (const std::bad_alloc& e) {
        std::cout << "bad_alloc: " << e.what() << "\n";
    }

    // std::bad_cast — неудачный dynamic_cast по ссылке
    Base b;
    try {
        Derived& d = dynamic_cast<Derived&>(b);  // бросает bad_cast
        (void)d;
    } catch (const std::bad_cast& e) {
        std::cout << "bad_cast: " << e.what() << "\n";
    }

    // По указателю — не бросает, возвращает nullptr
    Derived* d = dynamic_cast<Derived*>(&b);
    std::cout << "dynamic_cast по указателю: " << (d ? "успех" : "nullptr") << "\n";
}
```

## Производительность: happy path / sad path

**Happy path** — нормальное выполнение, исключений нет.
**Sad path** — исключение брошено и летит к `catch`.

**Happy path — практически бесплатно:** компиляторы используют zero-cost exceptions — `try` блок сам по себе ничего не стоит. Компилятор генерирует unwind tables в бинарнике, которые используются только при броске.

**Sad path — дорого:** поиск `catch` по таблицам, вызов деструкторов по пути.

Исключения — для **исключительных** ситуаций, не для управления потоком:

```cpp
#include <iostream>
#include <stdexcept>
#include <vector>

// Плохо — исключение как goto (антипаттерн)
int find_bad(const std::vector<int>& v, int target) {
    try {
        for (int i = 0; i < (int)v.size(); i++)
            if (v[i] == target) throw i;  // антипаттерн!
    } catch (int idx) {
        return idx;
    }
    return -1;
}

// Хорошо — обычный return
int find_good(const std::vector<int>& v, int target) {
    for (int i = 0; i < (int)v.size(); i++)
        if (v[i] == target) return i;
    return -1;
}

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    std::cout << find_bad(v, 3) << "\n";   // 2
    std::cout << find_good(v, 3) << "\n";  // 2
}
```

## Исключения в конструкторах

Если исключение вылетело из конструктора — объект считается **не созданным**, деструктор **не вызовется**.

```cpp
#include <iostream>
#include <stdexcept>
#include <memory>

// Плохо — голые указатели, утечка при исключении
struct FooBad {
    int* a;
    int* b;

    FooBad() {
        a = new int(1);
        std::cout << "a выделен\n";
        throw std::runtime_error("упс");  // ~FooBad не вызовется — утечка a!
        b = new int(2);
    }

    ~FooBad() {
        std::cout << "~FooBad\n";  // никогда не выведется
        delete a;
        delete b;
    }
};

// Хорошо — RAII, утечки нет
struct FooGood {
    std::unique_ptr<int> a;
    std::unique_ptr<int> b;

    FooGood() : a(std::make_unique<int>(1)) {
        std::cout << "a создан\n";
        throw std::runtime_error("упс");  // ~unique_ptr<int> для a вызовется!
        b = std::make_unique<int>(2);
    }

    ~FooGood() {
        std::cout << "~FooGood\n";  // тоже не выведется — объект не создан
    }
};

int main() {
    try {
        FooBad bad;  // утечка памяти для a
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n\n";
    }

    try {
        FooGood good;  // утечки нет — unique_ptr почистил a
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n";
    }
}
```

## Исключения в деструкторах

Деструкторы — `noexcept` по умолчанию с C++11. Если деструктор бросит исключение во время раскрутки стека — два исключения одновременно → `std::terminate`.

Если нужно обработать ошибку освобождения ресурса — делать это явно до деструктора:

```cpp
#include <iostream>
#include <stdexcept>
#include <cstdio>

struct File {
    FILE* f = nullptr;

    File(const char* name) {
        f = fopen(name, "w");
        if (!f) throw std::runtime_error("не удалось открыть файл");
        std::cout << "файл открыт\n";
    }

    void close() {  // явный метод — может бросать
        if (!f) return;
        if (fclose(f) != 0) throw std::runtime_error("ошибка закрытия");
        f = nullptr;
        std::cout << "файл закрыт явно\n";
    }

    ~File() noexcept {
        if (f) {
            fclose(f);  // last resort — ошибку игнорируем
            std::cout << "файл закрыт в деструкторе\n";
        }
    }
};

int main() {
    try {
        File f("test.txt");
        f.close();   // обрабатываем ошибку здесь если нужно
        // деструктор ничего не сделает — f == nullptr
    } catch (const std::exception& e) {
        std::cout << e.what() << "\n";
    }
}
```

## Исключения при конструировании полей, базовых классов, массивов

### Поля класса

Поля инициализируются в порядке объявления. Если одно поле бросило — все уже созданные поля уничтожаются:

```cpp
#include <iostream>
#include <stdexcept>
#include <memory>

struct Tracked {
    int id;
    Tracked(int id) : id(id) {
        std::cout << "Tracked(" << id << ") создан\n";
        if (id == 2) throw std::runtime_error("ошибка при создании Tracked(2)");
    }
    ~Tracked() { std::cout << "~Tracked(" << id << ")\n"; }
};

struct Foo {
    Tracked a{1};  // создаётся первым
    Tracked b{2};  // бросает
    Tracked c{3};  // не создаётся вообще
};

int main() {
    try {
        Foo f;
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n";
    }
    // вывод: Tracked(1) создан → Tracked(2) создан (начало) → ~Tracked(1) → поймали
}
```

### Базовые классы

```cpp
#include <iostream>
#include <stdexcept>

struct Base1 {
    Base1()  { std::cout << "Base1 создан\n"; }
    ~Base1() { std::cout << "~Base1\n"; }
};

struct Base2 {
    Base2()  { std::cout << "Base2 создан\n"; throw std::runtime_error("ошибка в Base2"); }
    ~Base2() { std::cout << "~Base2\n"; }  // не вызовется — не создан
};

struct Derived : Base1, Base2 {
    Derived() : Base1(), Base2() {}
    ~Derived() { std::cout << "~Derived\n"; }  // не вызовется
};

int main() {
    try {
        Derived d;
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n";
    }
    // Base1 создан → Base2 начал → ~Base1 → поймали
}
```

### Массивы

```cpp
#include <iostream>
#include <stdexcept>

struct Elem {
    static int count;
    int id;

    Elem() : id(count++) {
        std::cout << "Elem(" << id << ") создан\n";
        if (id == 3) throw std::runtime_error("ошибка на элементе 3");
    }
    ~Elem() { std::cout << "~Elem(" << id << ")\n"; }
};
int Elem::count = 0;

int main() {
    try {
        Elem arr[6];  // создаст 0,1,2,3 — на 3 бросит, 0,1,2 уничтожатся
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n";
    }
}
```

### Делегирующий конструктор

```cpp
#include <iostream>
#include <stdexcept>

struct Foo {
    Foo(int x) {
        std::cout << "Foo(int) — основной конструктор\n";
    }

    Foo() : Foo(42) {  // делегирует — после этого объект считается созданным
        std::cout << "Foo() — тело делегирующего\n";
        throw std::runtime_error("бросаем после делегирования");
    }

    ~Foo() {
        // вызовется! объект считается созданным после успешного Foo(42)
        std::cout << "~Foo вызван\n";
    }
};

int main() {
    try {
        Foo f;
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n";
    }
}
```

## Исключения при создании параметров функции (до C++17)

Порядок вычисления аргументов функции **не определён стандартом**. До C++17 компилятор мог чередовать вычисление аргументов (interleaving):

```
1. new A()          — выделили память
2. new B()          — бросает исключение!
3. unique_ptr<A>()  — никогда не выполнится — утечка памяти для A!
```

**Решение — `make_unique`:** одна атомарная операция, interleaving невозможен.

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>

struct A {
     A() { std::cout << "A создан\n"; }
    ~A() { std::cout << "~A\n"; }
};

struct B {
     B() { std::cout << "B создан\n"; throw std::runtime_error("ошибка в B"); }
    ~B() { std::cout << "~B\n"; }
};

void foo(std::unique_ptr<A> a, std::unique_ptr<B> b) {
    std::cout << "foo вызван\n";
}

int main() {
    // Безопасно с make_unique — каждый аргумент атомарен
    try {
        foo(std::make_unique<A>(), std::make_unique<B>());
    } catch (const std::exception& e) {
        std::cout << "поймали: " << e.what() << "\n";
        // A будет уничтожен корректно
    }
}
```

**C++17** — проблема решена: каждый аргумент вычисляется полностью до начала следующего. Но `make_unique` всё равно предпочтительнее.

## Почему нет `stack::pop()` возвращающего значение

`pop()` возвращает `void` — только удаляет. Получить значение можно только через `top()` до удаления. Это даёт **строгую гарантию**: либо всё прошло успешно, либо стек не изменился.

```cpp
#include <iostream>
#include <stack>
#include <stdexcept>

int main() {
    std::stack<int> s;
    s.push(1);
    s.push(2);
    s.push(3);

    // Правильный способ
    while (!s.empty()) {
        int val = s.top();  // сначала получаем
        s.pop();            // потом удаляем
        std::cout << val << "\n";
    }

    // Если бы pop() возвращал значение и копирование бросило исключение —
    // элемент уже удалён из стека, но не доставлен вызывающему. Данные потеряны.
}
```
