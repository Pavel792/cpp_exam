# Многопоточность

- **Синтаксис**  
```
#include <thread>
#include <iostream>

void printMessage(const std::string& msg) {
    std::cout << msg << std::endl;
}

int main() {
    // 1. Передача функции и аргументов
    std::thread t1(printMessage, "Hello from thread!");
    
    // 2. Лямбда
    std::thread t2([]() {
        std::cout << "Lambda thread\n";
    });
    
    // 3. Метод класса
    class Worker {
    public:
        void doWork() { std::cout << "Working...\n"; }
    };
    Worker w;
    std::thread t3(&Worker::doWork, &w);  // Указатель на метод и объект
    
    t1.join();
    t2.join();
    t3.join();
}
```
Лучше передавать аргументы по значению, чтобы получать их копии.

- **Гонки**  
```
#include <thread>
#include <iostream>

void unsafePrint(int id) {
    std::cout << "Thread " << id << " says hello\n";
    // Гонка! Несколько потоков пишут в cout одновременно
}

int main() {
    std::thread t1(unsafePrint, 1);
    std::thread t2(unsafePrint, 2);
    
    t1.join();
    t2.join();
}
```
Чтобы бороться с гонками, можно использовать `mutex` - кто захватил, тот и работает, потом отпускает. 
```
#include <thread>
#include <iostream>
#include <mutex>

// Глобальный мьютекс для защиты std::cout
std::mutex coutMutex;

void safePrint(int id, const std::string& message) {
    // Захватываем мьютекс перед использованием cout
    coutMutex.lock();
    
    std::cout << "Thread " << id << ": " << message << "\n";
    
    // Освобождаем мьютекс после использования
    coutMutex.unlock();
}

int main() {
    std::thread t1(safePrint, 1, "Hello");
    std::thread t2(safePrint, 2, "World");
    
    t1.join();
    t2.join();
    // Теперь вывод гарантированно цельный:
    // Thread 1: Hello
    // Thread 2: World
    // (или наоборот, но строки не перемешаются)
}
```

Вместо `mutex`, который можно забыть освободить, можно использовать  
`std::lock_guard<std::mutex> lock(mtx);`(освобождается при вызове деструктора). `lock_guard` нельзя освободить вручную, `unique_lock` можно.  

- **Атомарные операции**  
```
#include <atomic>

std::atomic<int> counter{0};  // Атомарный счётчик

void increment() {
    for (int i = 0; i < 100000; ++i) {
        ++counter;  // ✅ Атомарная операция, нет гонки данных
    }
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    
    t1.join();
    t2.join();
    
    std::cout << counter;  // Всегда 200000
}
```

- **Потокобезопасность cout**  
При обращении к std::cout не будет гонки, но вывод может смешаться.  

- **Joinable/detached потоки**  
Что делает join():  
Блокирует текущий поток, пока целевой поток не завершится.  
Очищает ресурсы потока.  
После join() поток становится не joinable.  
Что делает detach():  
Отсоединяет поток, позволяет ему работать независимо.  
Мы больше не можем взаимодействовать с этим потоком.  
После detach() поток не joinable.
