# Классы  

- **Синтаксис**  
Пользовательский тип данных, объединяющий данные (поля) и функции для работы с ними (методы).  
```
class MyClass {       // Объявление класса
private:              // Секция доступа (по умолчанию private в class)
    int data;         // Поле (член-данные)
    
public:               // Публичная секция (интерфейс)
    // Конструктор
    MyClass(int val) : data(val) {} // Список инициализации
    
    // Метод
    void print() const { // const метод не изменяет объект
        std::cout << data;
    }
    
    // Геттер
    int getData() const { return data; }
    
    // Сеттер  
    void setData(int val) { data = val; }
};
```

- **Поля**  
Обычные - инициализируются по порядку, могут быть неинициализированными.  
Константные - инициализируются только в списке инициализации.  
Ссылки - инициализируются только в списке инициализации, должны быть инициализированы.  
Порядок в списке инициализации не важен, в порядке объявления.  
Уничтожение в порядке, обратном порядку объявления.  

- **Копирование**  
`ClassName(const ClassName& other);` - копирующий конструктор.  
`ClassName& operator=(const ClassName& other);` - оператор копирующего присваивания.  

- **Конструктор**  
```
class MyClass {
public:
    // 1. Конструктор по умолчанию
    MyClass();
    
    // 2. Конструктор с параметрами
    MyClass(int a, double b);
    
    // 3. Копирующий конструктор
    MyClass(const MyClass& other);
    
    // 4. Перемещающий конструктор (C++11)
    MyClass(MyClass&& other);
    
    // 5. Делегирующий конструктор (C++11)
    MyClass(int a) : MyClass(a, 0.0) {}
};
```
**Конструктор по умолчанию** — конструктор, который может быть вызван без аргументов. Генерируется при отсутствии пользовательских конструкторов или при `= default;`.  
Для встроенных типов (int, float, указатели) — не инициализирует (оставляет мусор).  
Для членов-объектов с конструктором по умолчанию — вызывает их конструкторы.  
Для базовых классов с конструктором по умолчанию — вызывает их конструкторы.  
Вызывается при объявлении экземпляра класса без аргументов.  
**Делегирующий конструктор** вызывает другой, обязательно неделегирующий, конструктор.  

- **Иницализация полей**  
```
class Example {
    // 1. Инициализация в объявлении (C++11) - САМЫЙ ПРИОРИТЕТНЫЙ
    int a = 10;          
    std::string s = "default";
    
    // 2. Константы (можно без присваивания)
    static const int MAX = 100;
    
    // 3. Список инициализации конструктора
    Example(int val) : a(val), s("custom") {}
    
    // 4. Присваивание в теле конструктора (НЕ инициализация!)
    Example() { a = 20; } // a сначала = 10, потом перезаписывается на 20
};
```
Порядок срабатывания:  
Инициализация в объявлении (если есть).  
Список инициализации конструктора (может переопределить п.1).  
Тело конструктора (может переопределить всё).  
Лучше всегда инициализировать все поля.  

- **Default member initializer (C++11)**  
Способ инициализации членов класса непосредственно в месте их объявления внутри класса.  

- **Почему не надо использовать переприсваивание в конструкторе**  
Стоит использовать список инициализации, более оптимизировано + меньше проблем с некоторыми типами данных.  

- **Приватные/публичные поля и методы**  
private - доступ только внутри класса.  
protected - доступ внутри класса + наследники.  
public - доступ для всех.  
Чтобы обращаться не только к `this`, нужно передать ссылку на другого представителя.  
Чтобы изменять и читать приватные поля, придумали геттеры и сеттеры, чтобы проверять инвариант.

- **Отличия struct/class**  
В struct поля и методы по умолчанию публичные, в class - приватные.  

- **Вложенные классы**  
Вложенный класс — это класс, объявленный внутри другого класса.

- **Агрегатная инициализация**  
```
struct Point {
    int x;
    int y;
};

Point p = {10, 20};
```
Когда это работает:  
Нет приватных/защищённых членов.  
Нет пользовательских конструкторов (в том числе от пользователя).  
Нет виртуальных функций.  
Если аргументов будет меньше, стандартные типы будут обнулены, от других вызовется конструктор по умолчанию.

# Функции-члены (методы)  

- **Синтаксис**  
```
class MyClass {
private:
    int data;
    
public:
    // 1. Объявление метода внутри класса
    void print() {
        std::cout << data;
    }
    
    // 2. Константный метод (не меняет объект)
    int getData() const {
        return data;
    }
    
    // 3. Статический метод (не привязан к конкретному объекту)
    static void staticMethod() {
        // Не может обращаться к нестатическим полям!
        // std::cout << data; // ОШИБКА
    }
    
    // 4. Метод с параметрами
    void setData(int newValue) {
        data = newValue;
    }
    
    // 5. Только объявление, определение вне класса
    void externalMethod();
};

// Определение вне класса
void MyClass::externalMethod() {
    // реализация
}
```

- **. или ->**  
Точка при вызове от представителя класса. Стрелочка при вызове от указателя на представтеля.  

- **this**  
`this` - указатель на текущий объект, доступный внутри любого нестатического метода класса.  
Можно не писать при доступе к полям класса и вызове его же методов.  
Нужно явно писать для решения конфликта имён, возвращения ссылки на текущий объект и передачи в функцию.  
**Чем метод отличается от свободной функции?**   
Имеет доступ к private/protected полям своего класса.  
Неявно получает указатель this на текущий объект.  
Может вызывать другие private методы своего класса.   
**Захват this в лямбду** 5 разных случаев, стоит написать пример под все.  

- **Const correctness**  
Если метод объявлен как const, то this имеет тип const T*, и через него нельзя менять поля объекта.  
Можно менять через поля-указатели, потому что они в процессе не меняются.  
UB возникает при изменении const объекта.  

- **Определение методов внутри и снаружи класса**  
Внутри - неяный `inline`, снаружи без.  
`inline` делает сивол слабым и компилятор может выбрать любой, если определения идентичны. Если разные - UB.

#  Статические члены класса  

- **Статические поля**  
Объявлять лучше в `.hpp`, а определять в `.cpp` до первого использования именно переменной. Константные целочисленный можно определять прямо в классе.  
Статическое поле видно из любого класса, к нему можно обращаться изнутри, снаружи через namespace:: или представителя. В отличие от глобальных переменных не загрязняет глобальное пространство имён. 

- **Определение снаружи класса**  
Определять static поля нужно снаружи класса, const static можно внутри.

- **Отличия статических методов**  
Принадлежат классу, а не объекту, поэтому нет `this`. Имеет доступ только к статическим полям, поэтому остальные нужно передать.  

- **Линковка**  
При линковке static переменная становится сильным символом, поэтому работает ODR.  


#  Друзья и перегрузка операторов  

- **friend функции**  
Имеют доступ к приватным полям, в остальном обычные функции. Обычно объявляются в классе, а определяются после.  

- **Друзья классы**  
Имеют доступ ко всем полям и методам. Дружба односторонняя и не наследуется.
При друге в другом пространстве имён, просто указываем его через `::`, в сотальном то же самое.  

- **Перегрузка операторов**  
**Унатрые операторы** отличаются наличием `int` в аргуметах.  
**Бинарные операторы** часто принимают на вход другого представителя класса, и сравниваются с его полями.  

- **operator()**  
Оператор вызова функции.  

- **operator,**  
Возвращает ссылку на правый операнд, что гарантирует порядок вычислений. Если поменять, гарантия пропадёт. При перегрузке && и || они меняют порядок вычисления. 

- **operator+()**  
При вызове a + b, компилятор вызывает operator+() и может делать неявные преобразования. При вызове именно операторе без преобразований.  
Если не передавать значения в скобки, то вернет копию +-1.  

- **operator<=>**  
При `= default;` генерируте все опероторы сравнения. Сам по себе возвращает less, great или equal, умеет сравнивать простые типы.
